APIs

The ChangeGear REST API consists of multiple smaller APIs that expose different features of ChangeGear. These features are made available through endpoints, which are URLs that accept requests. For example, the Login API contains two separate endpoints: one for creating a user session and one for ending a user session.

This section explains what each endpoint does, how to access them, what parameters to pass, and what to expect from the REST API in response.
