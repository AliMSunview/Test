Integration

The Integration API allows external applications to communicate with ChangeGear. It is designed to be a hook for third-party issue tracking software to update ChangeGear when a ticket is modified. For more information on integration, see Administering ChangeGear > Advanced Customizations > Integrating with Third Party Issue Tracking Software.

To use these endpoints, add the full URL of the endpoint as a WebHook (or Service Hook) to your external application. Note that there are no body parameters for these requests; because this is simply a WebHook, the external application handles populating and formatting the contents of the request.

Tip: The Integration API supports basic authentication. For more information, please see Creating a Request > Headers.

Proxy Jira Requests

Jira does not support sending credentials via WebHooks. As a result, ChangeGear must proxy Jira calls through the Login endpoint before it can process incoming requests. However, this requires some additional configuration.

To configure the Jira proxy:

  1. Encode the credentials of the ChangeGear account that you wish to use for the integration. To do this, you will need to take your username and password separated by a colon (e.g. cguser:cgpassword) and encode it using Base64. For more information, see the Headers section under Creating a Request.

  2. Log into the server hosting the ChangeGear server.

  3. Navigate to the ChangeGear server installation directory and run the EncryptionPwd.exe file (defaults to C:\Program Files (x86)\SunView Software\ChangeGear\Server\EncryptionPwd.exe).

  4. In the Password field, enter your Base64 encoded credentials. Repeat this previous step for the Confirm Password field, then click Generate.

  5. Copy the contents of the Encrypted Password field. You will need this for later.

  6. Log into the server hosting the ChangeGear REST API.

  7. Open the REST API's web.config file (defaults to C:\Program Files (x86)\SunView Software\ChangeGear\CGRestAPI\Web.config).

  8. In the <appSettings> section, scroll to the line beginning with <!--add key="CGJiraCreds". Uncomment the line by removing the "<!--" and "--" characters.

  9. Replace the contents of the value property with the Encrypted Password that you copied earlier. Make sure that your pasted contents are surrounded in double quotes.

  10. Save the file, then perform an IIS reset or recycle the CGRestAPI application pool.
