Using the REST API

The ChangeGear REST API provides access to ChangeGear services through a RESTful interface. It allows you to interact with ChangeGear entities using simple HTTP requests. The REST API opens up the possibility to programmatically access and modify ChangeGear using scripts, custom commands, and third-party services.

Important: We strongly recommend using SSL to secure communications to and from your ChangeGear server.

This section explains how to use the REST API, including:

  • How to format a request

  • How to parse the response

  • How to perform authentication

  • How to interact with entities (tickets, announcements, Knowledge Base articles, etc.)

RESTful Services

REST (or Representation State Transfer) is an architectural approach to communicating data between a client machine and a server. Services that implement REST are known as RESTful services. While REST does not define a standard, RESTful services often share the following characteristics:

  • REST services are stateless. Each transaction between the client and server is treated as a completely separate transaction. The client provides all of the information that the server needs to service each request.

  • REST services are uniform. Each request consists of the same components: the location of a resource, the action to take on the method, and any additional information necessary to complete the request.

  • REST services are scalable. RESTful services can grow to support a large number of components–and methods for interacting with those components–without negatively impacting performance or usability.

A typical REST interaction consists of a request and a response. Requests originate from client machines wishing to retrieve data from a server. When the server processes the request, it provides a response that the client then further processes before presenting to the user.
