Creating Grid Tables

Grid tables allow you to create custom tables for storing lists of entities. Once you create a table, you can:

  • Define the fields and data types for the grid table

  • Define the end user and staff forms for the grid table

  • Create a CustomGridControl control to display the contents of the grid table in a grid on your module's form

  • Add, edit and delete grid table entries

For example, you could create a grid table in the Change module to track risks associated with each request. Each risk might consist of a name, a brief description of the risk, and a severity level.

To create a new grid table:

  1. Access the Entity Editor for the correct module.

  2. Select the Grid Tables node and click Add Entity.

  3. Enter a Name for the new grid table. This represents the internal name of the grid table. Optionally, enter a Caption to set the display name of the grid table.

  4. Expand the Model node to add fields and associations to the grid table. For more information, see How Do I Add a Field to an Entity? and How Do I Add an Association? respectively.

  5. Save the model by clicking File > Save All Changes. This also generates a new default staff form for the grid table.

    1. Optionally, configure the entity's default form or add your own. If you modified the form, save the model again by clicking File > Save All Changes.

In order to use your grid table in a module, you must first create an association, (test comment) then add that association to a CustomGridControl inside of the form. For more information on creating an association, see How Do I Add an Association?.

To add a grid table to a form:

  1. In the module's main entity model, create an association to the grid table by expanding the Module > Module Entity > Model > Associations nodes. For example, if you created your grid table in the Incident module, click Incident Management > Incident Request > Model > Associations.

  2. For the FieldDataType property, select the name of your grid table.

    Tip: The name will contain I_UDT_ followed by the name that you specified in the grid table's Name property.

  3. In the FieldName property, enter the name of the association as it will appear in the Fields section of the Entity Editor. Optionally, you can configure the association further by expanding the FieldAttributes node.

  4. Save the model by clicking File > Save All Changes.

    Tip: If you see the error "The column 'AssociationName1Name' already exists", expand the MappedColumns property of the association by clicking the button and change the MappedColumnName property.

  5. Open the Form Designer to the form you wish to add the grid table to.

  6. From the Designer Toolbox, drag and drop a new CustomGridControl on to the form. Edit these properties as necessary:

    1. Set the FieldName property to the name of the association you created, beginning with "UDF_" (e.g. UDF_MyGridAssociation).

    2. Set the TableName property to the name of the grid table you created, beginning with "I_UDT_" (e.g. I_UDT_MyGridTable).

    3. Set the ShowFilter property to true if you want to display a text box for filtering columns in the Web client.

    4. Set the ShowGroupBy property to true to allow users to group by one or more columns in the Web client.

    5. Set the View property to the name of the view that you wish to use for the grid table. For now, enter Default. For more information on creating custom views, see Creating and Editing Views.

  7. Save and activate the form.

To ensure that your changes propagate, restart the ChangeGear service and perform an IIS reset.
