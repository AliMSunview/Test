Creating Lookup Lists

Lookup lists are ChangeGear entities with data that is controlled by the ChangeGear administrator instead of by end users. Lookup lists are generally displayed as drop-down controls on forms. When you create a lookup list, you create a new entity and then create a form that will be used to populate the lookup list in the Desktop Client. After you have created the lookup list and saved your changes, you can add it to a form on the Web.

There are five steps to creating and using a lookup list:

  1. Create the lookup list in the Entity Editor

  2. Add the lookup list as an association to the ticket entity (i.e., Incident Request, Problem Report, or RFC)

  3. Modify the lookup list form in the Entity Editor

  4. Add the lookup list to a form as a field in the Form Designer

  5. Add data to the lookup list in Administration > Module > Lookup Lists

Note: Lookup list items can only be added through the respective Administration > Module > Lookup Lists node.

The following example guides you through creating a lookup list and viewing the lookup list outside of the Entity Editor. The result of this example is a new lookup list added to the Change module called "Service Request Type".

To create a lookup list:

  1. Access the Entity Editor for the correct module. In this example, select Change Management > Lookup Lists.

  2. Click Add Entity. The New Entity dialogNew Entity dialog will display.

  3. Enter the name of the lookup list. The name is how the lookup list is stored in the database. In this example, enter "ServiceRequestType."

    Note: The name cannot include any special characters, including spaces and underscores.

  4. Enter the caption of the lookup list. The caption is a more human-readable way to refer to the lookup list. In this case, enter "Service Request Type."

  5. Click OK. The new lookup list is added to the tree.

  6. The Model node defines the fields and associations that are available to each entry in the lookup list. Fields contain individual data points, while associations allow you to create links between lookup lists and other data sources or other lookup lists. For more information on adding and configuring fields and associations, see How Do I Add a Field to an Entity? and How Do I Add an Association?.

  7. Save your changes to the entity model by clicking File > Save All Changes. It may take several minutes to save the changes as ChangeGear is updating the database with new tables and relationships. A default form will be automatically generated when the model has been successfully saved.

  8. The Forms node defines the form used to enter items into the lookup list. The Default (Staff) form can be accessed by navigating to the module's respective Administration tab > Module > Lookup Lists node and adding or editing a lookup list item.

    1. Click Edit Form or the Customize... button. The Form Designer displays.

    2. Click Design Mode.

    3. Add and arrange fields as necessary. Custom fields are preceded by "UDF_". For example, if you created a custom field named "Description", the field will be listed as "UDF_Description". For more information on configuring forms, see How Do I Add a Field to a Form?.

    4. Save your form.

  9. Save any changes to the model by clicking File > Save All Changes.

  10. Restart the Internet Information Services (IIS) process or recycle the CGWeb application pool to refresh the ChangeGear Web client.

Before you can use the lookup list in a Web form, you need to add the lookup list to the entity's model as an association. For more information, see How Do I Add an Association?.

To view the new lookup list:

  1. Close the Entity Editor.

  2. In the main Desktop Client window, use the Administration tree to navigate to the module in which you created the lookup list.

  3. Expand Lookup Lists.

The new lookup list should now be displayed with the other lookup lists. For information regarding adding entries to a lookup list, see the Lookup Lists chapter.

Using Lookup Lists with Associations

Lookup lists display a list of values that users can choose from. However, these values are manually defined in the lookup list itself and have to be changed in the Desktop Client. Using associations, you can link a lookup list to another entity in order to create dynamic and flexible lookup lists. This creates a relationship where one entity becomes a lookup list item for another entity. You can then bind the association to a field on a form to allow users to select and even modify values in the lookup list from the ChangeGear Web client.

For more information on using associations, see the How Do I Add an Association? topic under Customizing the Model .
