Using the Entity Editor

By using the Entity Editor, you can change ChangeGear's underlying data model and the forms used to modify individual data items. The editor is divided into modules that generally correspond to the module tabs in ChangeGear for easy navigation. Once you open the module, the editable entities are displayed, including any new lookup lists you create. For example, if you open the Change Management module, the editable entities include the fields used to store data about RFCs, the forms used by the Web client to modify those fields, and any custom lookup lists or grid tables used in conjunction with RFCs.

Important: SunView Software highly recommends creating a backup of your ChangeGear database and installation folder on a test system before using the Entity Editor.

Within the Entity Editor, you can access the following utilities:

Form Designer – The Form Designer is a drag-and-drop editor used to create and modify forms. Each form consists of controls, which are elements used to display data from or submit data to a field. You can have multiple forms saved, but only one can be marked as active (see Dynamic Request Automation for information regarding using multiple forms). The Form Designer provides a default set of controls based on commonly-used fields, but you can also configure controls for added flexibility. See Customizing Forms for more information.

Model Editor – The Model Editor allows you to modify fields and create associations with other entities. Changes made at the module level take precedence over other changes. See Customizing the Model for more information.

Lookup List Editor – The Lookup List Editor lets you define new lookup lists for a module. Each lookup list has its own form (used when adding items to the lookup list), fields, and associations. Note that you can only create new lookup lists in the Entity Editor. If you wish to add items to a lookup list, see Adding a Lookup List Item.

Grid Table Editor – The Grid Table Editor lets you create new grid views. Each grid table has its own form (used when adding items to the grid table), fields, and associations. For more information, see Creating Grid Tables.

Use the Entity Editor to:

  • Add fields to forms

  • Create new attributes

  • Modify default properties of attributes

  • Change the captions of an attribute

  • Modify the appearance of a form

  • Activate forms

  • Create grid views

  • Revert attributes and forms to the factory default

The Configuration Management module is covered separately as there are some additions made specific to the CMDB. For more information, see Extending the CMDB.
