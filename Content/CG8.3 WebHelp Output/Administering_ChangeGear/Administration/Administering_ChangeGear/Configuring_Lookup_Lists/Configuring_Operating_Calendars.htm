Configuring Operating Calendars

Operating Calendars are used to configure time tracking options specific to an SLA. When creating or editing an SLA, you can use an Operating Calendar to define the business hours that the SLA will follow. While you can only have one Operating Calendar per SLA, you can have separate Operating Calendars for separate SLAs. If you have teams with varying business hours, Operating Calendars make it easy to independently track each team's time.

Operating Calendars can also use holiday schedules to pause the SLA clock on certain days. Each Operating Calendar can be assigned one holiday schedule. For more information, see Configuring Holiday Schedules.

When applied to an SLA, an Operating Calendar overrides the Global Business Hours setting.

To create a new Operating Calendar:

  1. Click the Administration tab.

  2. In the Administration section, click General.

  3. Click Operating Calendars.

  4. Under Common Tasks, click New. The Operating Calendar Details dialog appears.

  5. In the Name field, enter a name for the new calendar.

  6. In the Description field, enter a description of the new calendar.

  7. If the calendar is used primarily in a different time zone, select the Time Zone from the drop-down list. ChangeGear takes time zones into effect when calculating the expiration date of an SLA.

    Note: The Time Zone field defaults to the server time zone.

  8. Select the Week Days that the calendar will follow by using the check boxes. The default is Monday through Friday.

  9. Select the Daily Hours that the calendar will follow by using the up and down arrows, or by entering the time in the text boxes. Select the 24 hours check box if the SLA clock should run for the entire day.

  10. Optionally, select a Holiday Schedule from the Holidays drop-down list. For more information on Holiday Schedules, see Configuring Holiday Schedules .

  11. Click OK to close the Operating Calendar Details dialog.
