Manual Creation with Automated Publication

ChangeGear can be configured to publish manually created announcements automatically using the Business Policy Automation feature. Announcements published this way are usually reviewed to ensure that the appropriate tone and level of detail for the audience has been applied. For example, if your company has a communication plan that notifies impacted users during certain steps of the change process, you can create multiple announcements for each phase of the change process, and you can automate the publication of the announcements.

To create a Business Rule that automatically publishes an existing announcement:

  1. In the Administration module, navigate to Announcement Management > Business Policy Automation.

  2. In the Common Tasks menu, click New. The Business Rule DetailsBusiness Rule Details dialog box is displayed.

    • In the Name field, type a name for the rule.

    • In the Description field, type a brief description of the rule.

    • The Enable this rule check box is selected by default. If you don't want the rule to be active, clear the check box.

    • Specify how often or when you would like ChangeGear to check for conditions matching your Business Rule.

    • In the MaxOccurs box, specify how many times you want the rule to execute per item (Tasks or RFCs, for example).

      Note: Click Execute to run a rule immediately.

  3. Click the ConditionsConditions tab.

    • Use the lists to specify condition(s) that, when met, will trigger the Business Rule.

    • Choose the field Status from General and the appropriate operator and value.

    • Select Auto-Publish Status and the appropriate operator and value.

    • Select a time period and the appropriate operator or operator and value pair.

  4. In the Actions tab click New to create an action that executes when triggering the rule.

  5. Select Edit Value in the Create an Action dialog. The Announce DetailsAnnounce Details dialog displays.

    • Enter the name of the action in the Name field.

    • Enter the description in the Description field.

    • In Field Name, choose Status.

    • In the Value field, choose Published.

      Note: You must first choose a Field Name before you can choose a Value.

    • Click Ok.

  6. Click Ok in the Business Rule Details dialog.

  7. Either open or create an announcement.

  8. In the DetailsDetails tab of the announcement, choose a value for Publish when RFC is.

  9. Save your changes and close the announcement.

According to the choices made in the images, each announcement set to “Publish when RFC is Assigned” that has been reviewed and is scheduled to begin this week will be published automatically when the RFC’s status is pending approval.

This method creates a Business Rule which automatically publishes a manually created announcement. In addition to conditions you specify in the rule, the Publish when RFC is field in the Announcement Details tab and the RFC’s status are used in this method.

Other ways to use this method include:

  • Automatically announce approved changes to users outside of your organization

  • Automatically announce completed changes to end users

These are just a few typical scenarios. Given the flexibility of ChangeGear's Business Policy Automation feature, there are many other possibilities. You could combine the two methods. For example, your organization could automatically create and publish change announcements to internal IT staff, while external communications, regardless of urgency, could rely on manually created announcements that are automatically published by combining these approaches.
