Creating Approval Groups

The Approval Groups feature allows you to control which users approve tickets based on configurable conditions - such as Impact, Urgency, and Priority - or you can create your own criteria for approvals. The approval groups are available for Change Request and Service Request tickets.

Important: If you have created multiple approver groups, view Determining Approvers to learn how approval groups are applied.

To create an Approval Group:

  1. Open the Administration module.

  2. In the Administration module, navigate to Change or Service Request module, e.g., Change Management > Approval Groups. The Approval Groups window displays.

  3. In the Common Tasks group menu, click New. The Approval Group Details dialog box displays.

  4. In the Name field, type the name of the new approval group.

  5. In the Description field, type a brief description of the new approval group.

  6. Select the criteria applied for a ticket to determine when to use an approval group.
    If you select ticket Type, go to Step 7,

    -OR-

    If you select Criteria, go to Step 8.

  7. Click the Select button and choose a ticket type. The new change type displays under Selected Types. You can expand the ticket Type to reveal sub-types using the down arrow. You can select multiple ticket types. To remove a ticket type, click the appropriate type under Selected Types, and click Remove.

    Note: Sub-types also uses this group unless you define a specific group for the sub-types, i.e., if you choose Facilities, the sub-types of Power connection, Lighting, and UPS automatically use the same approval group as Facilities. However, if you choose Power connection, only tickets with the ticket Type set as Power connection apply the approval group.

  8. Click the Criteria... button and enter the specific criteria used in determining which tickets require the groups approval. See Using the Criteria Editor for more information.

  9. Click OK to save.

  10. Under Approvers, click Add. The Select Approvers dialog box displays.

  11. From the Users tab, select the users you want to add to the approval group.

    - OR -

    From the Teams tab, select the teams you want to add to the approval group. Verify the team has members, each member will be part of the approval group.

    - OR -

    From the Roles tab, select the roles you want to add to the approval group.

    - OR -

    From the Dynamic Roles tab, select the roles you want to add to the approval group.

    Note: You can add any combination of Users, Teams, Dynamic Roles and/or Roles as approvers; however, only one type can be added at a time.

  12. Click OK. The designated approvers are shown in the Approval Group Details dialog.

  13. Group, sequence, or create conditions for the approvers as necessary.

  14. Click OK.
