Specifying Conditions

You can use conditions such as ticket impact, urgency and priority to determine which tickets an approver receives for approval.

To specify conditions:

  1. Click Administration tab.

  2. In the Administration module, navigate to Change or Service Request module, e.g., Change Management > Approval Groups. The Approval Groups page displays.

  3. Click the Approval Group you want to edit, and click Edit in the Common Tasks menu. The Approval Group Details dialog box displays.

  4. In the Approvers list, select the appropriate approver, and click Conditions. The Approval Condition Details dialog box displays.

  5. Click the Priorities tab and select the change priorities you want the selected approver to approve. You can also deselect priorities from which you want to exclude the approver.

  6. On the Impacts tab, select the change impacts you want the selected approver to approve. You can also deselect impacts from which you want to exclude the approver.

  7. On the Urgencies tab, select the change urgencies you want the selected approver to approve. You can also deselect urgencies from which you want to exclude the approver.

  8. Choose whether you want the approver to be included if all conditions must be met or if any of the conditions are met.

  9. Click to select the Every user with the same... uncheck the box if you want to ensure that every user with the same order number (number 2, for example) must approve the request.

  10. Click OK.
