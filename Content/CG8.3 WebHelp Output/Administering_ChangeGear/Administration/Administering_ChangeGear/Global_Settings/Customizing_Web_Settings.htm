Customizing Web Settings

The Application Settings section is where you can specify the URLs of core ChangeGear services. By default, these fields will be automatically populated using the name of the ChangeGear server. If any of these services have been installed on a different machine, then the URL will need to be changed to reflect this. If these URLs do not point to the correct server, certain features may not work.

After modifying any value in the Application Settings section, click Save to persist your changes.

Web Settings

The Web URL defines the location of the ChangeGear Web Application. This is the URL that end users and staff users use to access ChangeGear.

Note: You can also double click on the Web URL to open the ChangeGear Web application.

REST API Settings

The ChangeGear REST API exposes certain functions of the ChangeGear server through a RESTful API. Certain ChangeGear Web features (such as the Change Log, Field Recommender, and Meeting Notes) rely on the REST API in order to function properly. For more information on the REST API, see Using the REST API.

Intelligence Pack Settings

The Intelligence Pack is a ChangeGear component that provides features such as suggested solutions for tickets, intelligent assignment of tickets to users, and auto-filled form values based on similar tickets. The Intelligence Pack REST API URL defines the URL that ChangeGear uses to communicate with the Intelligence Pack backend service.

Important: Please see the separate Installation Guide document for more information on installing the Intelligence Pack.

Tip: Features in the ChangeGear Web client that utilize the Intelligence Pack can be identified by the Intelligence Pack icon shown above.

To enable and configure the Intelligence Pack:

  1. Click the Administration tab.

  2. In the Administration section, click Global Settings.

  3. Scroll down to the Intelligence Pack section.

  4. Click the Enable Intelligence Pack check box.

  5. In the URL of Intelligence Pack REST API text box, enter the URL and port number that the service is listening on. This defaults to port 10210 of the machine that the Intelligence Pack was installed on.

    Tip: If you cannot connect to the service, make sure that the firewall on the machine hosting the service has the correct port opened (defaults to 10210).

  6. Click Save, then close and reopen the ChangeGear Desktop Client.

  7. Restart IIS, or recycle the CGRestAPI application pool.
