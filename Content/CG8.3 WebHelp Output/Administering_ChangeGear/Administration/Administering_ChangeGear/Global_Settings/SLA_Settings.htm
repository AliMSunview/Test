SLA Settings

Service Level Agreements (SLAs) are contracts between a customer and a provider. SLAs administered from the Global Settings only impact new Incidents, Service Requests and RFCs entered in the system—SLAs do not affect items in the CMDB or the Service Catalog. Administering SLAs for the CMDB is accomplished using the lookup lists in the Configuration Management node of the Administration tree.

Creating and managing SLAs is accomplished via the SLA Manager in the ChangeGear Web client. The SLA Manager lets you configure SLAs per module. When SLAs are enabled, the SLA Manager can be accessed from the Other menu in supported ticket modules.

To enable SLAs:

  1. Open the Administration module.

  2. In the Administration section, click Global Settings.

  3. Scroll down to the SLA Settings section.

  4. Select Enable SLA.

    Warning: All activated SLAs will begin running as soon as you save your changes. This includes the factory SLA, which applies to all tickets.

  5. Click Save.

When using SLAs, you should also have notifications enabled and ensure that users have an active e-mail address associated with their account. You may also wish to review the Pause Clock and Recalculate SLA options available via the Workflow editor. For a complete explanation of SLA configuration and processing, refer to the ChangeGear Web Help under Administration > SLA Management.
