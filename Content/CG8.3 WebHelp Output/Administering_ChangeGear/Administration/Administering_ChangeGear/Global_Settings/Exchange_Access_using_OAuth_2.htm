Exchange Access using OAuth 2.0

ChangeGear can be configured to access Exchange Online mailboxes using OAuth 2.0. This can be used for Outbound Notifications from ChangeGear and Email Connector. 

To do this:

  1. An app needs to be set up inside Azure via the Azure Portal with the appropriate permissions and access.

For details on the steps to configure OAuth, refer to the separate document provided with the 8.3 release: ChangeGear 8.3 Using OAuth 2.0 to Access Exchange Online from within ChangeGear.

  1. Once the app is configured in Azure, copy the Application ID, Tenant ID, and Client Secret from the app and set it up inside ChangeGear as shown below.

  2. In the ChangeGear Desktop Client Admin Global Settings page, locate EWS OAuth Settings  and complete the following fields.

    

  1. Enter the Application (Client) ID,  Directory (Tenant ID), and Client Secret  that you received when setting up the Application.

  2. Click Save at the bottom of the page. 

Note: OAuth must also be selected in Notification Settings. See Configuring Notification Settings.
