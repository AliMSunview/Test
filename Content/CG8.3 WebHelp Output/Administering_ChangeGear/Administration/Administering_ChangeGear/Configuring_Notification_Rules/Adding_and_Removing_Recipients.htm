Adding and Removing Recipients

One of the ways you can make an exception to an existing notification rule is to change its recipients. A notification rule may not, for instance, include the recipients that you want to notify.

To add recipients to Notification Rules:

  1. In the Administration explorer, navigate to Notification Rules in the management node you wish to modify.

  2. In the Notification Rule list, select notification rule to which you want to add a recipient.

  3. In the Tasks group menu, click Edit.

  4. Click the Add button next to the E-mail or SMS Settings Recipients box.

  5. In the Type Name or Select from List field, type the name or select the name from the list box,

    - OR -

    Open another tab and select the Team, Role, Dynamic Role, or Contact.

    Note: If a ticket or item is owned by a team, the Security Team roles take precedence over user roles. For example, if one of the recipients is not part of the team, the recipient will only have the privileges defined in the Guest Role of the Security Team. If the Guest Role does not include privileges to view tickets or items, the recipient will receive an error when opening the ticket or item.

  6. Click Add to add the recipients to the list of Selected Recipients.

  7. Click OK.

Another notification rule may include recipients that you do not want to notify. Each change request lets you remove recipients from its rules.

To remove recipients from Notification Rules:

  1. In the Administration explorer, navigate to Routing/Notification > Notification Rules.

  2. In the Notification Rules list, select the notification rule to which you want to remove a recipient.

  3. In the Tasks group menu, click Edit.

  4. In the Recipients box, click on to highlight the recipient that you want to remove.

  5. Click Remove.

  6. Click OK.
