Administering ChangeGear

The Administration module lets you customize, configure, and adapt ChangeGear to your organization’s requirements. The Administration section provides the following modules for configuring and managing ChangeGear:

  • Security

  • Global Settings

  • Lookup Lists

  • Views

  • Notification Rules

  • Business Policy Automation

  • Workflows

  • Communication Templates

  • Approval Groups

  • Advanced Tools

  • Surveys

  • Knowledge Base Articles

  • Tasks

  • Announcements

  • Advanced Customizations

  • Activating New License Files

Applying Changes

In most cases, any changes made in the Administration module are immediately applied to ChangeGear upon save. However, some changes will not apply until the ChangeGear service or Internet Information Service (IIS) is restarted. Some of these changes include modifying URLs, modifying forms, or changing the elements in a lookup list. Instructions on restarting the ChangeGear and IIS services are listed below. If a service refresh or restart is required, it will be mentioned as a step in the configuration instructions.

Warning: Restarting the ChangeGear service while users are actively using the system may result in data loss. Please schedule a maintenance window if you are planning to restart a live system.

To restart the ChangeGear service:

  1. Log in to the Windows server where the ChangeGear server is installed.

  2. Open the Services snap-in by opening the Start menu and typing Services.

  3. Click ChangeGear Service, then click Stop the service.

- OR -

  1. Open the ChangeGear Desktop Client and log in as a ChangeGear administrator.

  2. Click the Administration tab, then click Global Settings in the navigation menu.

  3. Scroll to the Restart ChangeGear section, then click Restart.

To restart Internet Information Services (IIS):

  1. Log into the Windows server where ChangeGear Web is installed.

  2. Open the Internet Information Services (IIS) Manager by opening the Start menu and typing IIS.

  3. Select the node for the local server.

  4. In the Actions pane, click Restart.

As an alternative to restarting the entire IIS server, you may only want to restart the CGWeb or CGRestAPI application pools. Restarting an application pool is often faster than restarting the entire server and allows other web services to continue running, while providing the same effect.

To refresh or recycle an Application Pool:

  1. Perform steps 1-3 in the restarting Internet Information Services (IIS) instructions above.

  2. Expand the node for the local server and select Application Pools.

  3. Select either the CGRestAPI or CGWeb application pool depending on which service you wish to refresh.

  4. In the Actions pane, click Recycle.
