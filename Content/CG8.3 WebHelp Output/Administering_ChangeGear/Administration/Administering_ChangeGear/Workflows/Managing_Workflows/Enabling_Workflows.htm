Enabling Workflows

Before a workflow can take effect, it first has to be enabled or set as the default workflow. Enabling a workflow allows you to perform actions such as assigning the workflow to a ticket or defining role privileges. Setting a workflow as the default workflow means that any tickets that do not have a workflow assigned to them will automatically use the default workflow.

Note: For a ticket to use an enabled workflow, that workflow must be assigned to the ticket. You can learn more about assigning workflows to tickets in the Configuring Multiple Workflows topic under Leveraging Workflow Automation.

By default, all tickets use the default workflow. If a ticket is still open when the default workflow changes, then the ticket will immediately begin using the new workflow. This may lead to unexpected errors if, for instance, a ticket is in a State that was present in the old workflow but not the new workflow. However, assigning a workflow to a ticket means that the ticket will always use the assigned workflow unless it is disabled or deleted, regardless of what the default workflow is.

Enabling a workflow takes effect immediately. ChangeGear does not need to be restarted, nor do users need to log out of the ChangeGear Web client.

WARNING: Always set a workflow's privileges before assigning it to a ticket. Serviceaide recommends configuring privileges immediately after enabling a new workflow. You should never set a workflow as the default without first reviewing its privileges.

To enable a workflow:

  1. Open the Administration module.

  2. Open a management node and navigate to Workflows.

  3. Select a workflow.

  4. Click Enable from the common tasks menu.

To set a workflow as the default workflow:

  1. Open the Administration module.

  2. Open a management node and navigate to Workflows.

  3. Select a workflow.

  4. Click Set as Default from the common tasks menu. This also enables the workflow.

    WARNING: Always review a workflow's privileges before making it the default workflow. This ensures that access is granted to the workflow before it takes effect.
