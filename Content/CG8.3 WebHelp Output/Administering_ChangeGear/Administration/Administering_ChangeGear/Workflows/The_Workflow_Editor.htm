The Workflow Editor

Workflows impact everything about the life cycle of a ChangeGear entity from the actions that are available to the state is displayed in the timeline. Workflows dictate which actions are displayed, what actions are allowed, what states can be reached from another state, what is displayed in the timeline, and what default actions to take when another action is performed. Using the workflow editor, you can customize ChangeGear to more closely match your organization’s process and streamline tickets through the system.

Note: You may not have access to this ChangeGear feature depending on your permissions or ChangeGear edition.

The following screenshot depicts Actions and the Workflow Status Bar from the Change Management factory workflow.

Workflows are created in ChangeGear by modifying existing workflows. This document gives a brief overview of the workflow editor’s interface and how to use the editor. It then gives specific scenarios and instructions to accomplish those scenarios while providing instructions on managing and administering workflows.

Before you begin, take a moment to review the images and terms below to understand how these terms are used within ChangeGear.

Figure 2: State Diagram

State: A state is a stage in the behavior of an item or ticket in ChangeGear. States have four types—New, Process, Approval, and End. A new state has no incoming connections or states while an end state has no target connections or outgoing states. Process states have both incoming and outgoing/target connections and states. Approval states are a special type of state that must have outgoing states attached to Approve and Reject, and can have approval groups assigned to them. In the image above, “Unsubmitted” and “New” are both states. The state “Unsubmitted” is of the type New.

Action: An action triggers a transition to move an item or ticket from one state to another. Actions in ChangeGear can also trigger automated actions, such as notifications or announcements. New actions create new privileges that you must enable per role.

Transition: A transition is a progression from one state to another, triggered by an action.

Factory Workflow: A factory workflow is a workflow that is shipped with ChangeGear. Factory workflows may not be deleted or edited; however, they can be modified by saving the factory workflow as a different workflow.

Figure 3: ChangeGear Connections

Inbound Connection: An inbound connection is an action that connects to the selected state. In the example image above, “Submit” is an inbound connection to the “New” state from the “Unsubmitted” state.

Target Connection: A target connection is an action available to progress from one state to another state. In the example image above, “Submit for Approval” is a target connection from the “New” state to the “Pending Approval” state.

Figure 4: ChangeGear States

Inbound state: An inbound state is a state that leads to another state. In the example image above, “Unsubmitted” is the inbound state of “New.”

Outbound state: An outbound state is a state that leads from another state. In the example image above, “Pending Approval” is the outbound state of “New.

Workflow Status Bar: The workflow status bar is a graphical representation of the progression through the workflow. Changes made to the workflow may not be apparent in the workflow status bar unless specifically configured to be so.
