Defining Role Privileges

You can grant or deny privileges to your users from the RolesRoles module in the Administration section of ChangeGear. Privileges are managed on a per-workflow basis. If a module has multiple workflows enabled, you can select each workflow individually from the drop-down. Selecting the name of the module lets you set privileges for the module's default workflow.

Note: Selecting the module name exposes additional privileges, such as Access Module and View. Disabling these privileges may prevent users from accessing the module in the ChangeGear Web client.

To define Role privileges:

  1. Click Administration > Security > Roles.

  2. In the Roles module, double-click on the role that you want to modify.

  3. In the Role Details dialog, click the Privileges tab.

  4. In the Objects group, click the name of the object for which you want to define privileges. For modules with multiple enabled workflows, each enabled workflow is displayed under the name of the module. Clicking on the name of the module itself allows you to modify privileges for the default workflow, as well as privileges that affect the entire module. For more information on managing the state of workflows, see Enabling Workflows.

    WARNING: Always review a workflow's privileges before making it the default workflow.

  5. In the Privileges group, click the check box to enable each privilege you want to enable or disable for the role.

  6. Click OK to save your changes.
