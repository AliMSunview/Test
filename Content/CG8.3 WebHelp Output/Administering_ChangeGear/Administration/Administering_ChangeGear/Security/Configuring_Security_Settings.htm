Configuring Security Settings

In the Security Settings section, you can enable the automatic registration (auto-registration) of users and select a default user role. Auto-registration automatically creates accounts for users when they attempt their first ChangeGear login.

There are two methods for auto-registration. The first, Universal, will auto-register new users to a specific role of your choosing. Universal is very straight-forward: if the user attempting to login doesn't yet exist in ChangeGear, they are added automatically and assigned to the role you specify. The second method, Group-based auto-registration, is more dynamic than Universal. Group-based registration uses security groups within your Windows domain directory to dynamically assign a user to a role.

By default, Group-Based auto-registration attempts to find a user in one of the network security groups. If the user is found in one of the security groups, they are auto-registered to the corresponding role. If the user is not in one of these security groups, the user is not registered and is notified that he/she does not have permission to log in.

Neither methods of auto-registration have any impact on Teams as they are created exclusively within ChangeGear. Moreover, members of Security Teams must have their role within the team specifically assigned to them, which is independent of their user role in ChangeGear.

Network security groups need to be created by you or your network administrator. These groups can have an explicit list of users in them directly, or can consist of existing groups you may already have. For example, you may have an existing domain security group that lists your IT staff members. If you do, you can add that group to the "ChangeGear_IT Staff" group rather than adding each IT staff member directly to "ChangeGear_IT Staff."

Note that not all ChangeGear roles take part in group-based auto-registration, but the roles that typically contain the majority of users will use this feature. For example, the Change Manager role is not auto-registered; you will need to manually configure a user for this role.

To configure security settings:

  1. Open the Administration tab.

  2. In the Administration section, click Global Settings.

  3. Scroll down to the Security Settings section.

  4. To enable auto-registration, click the Auto-Registration Enabled checkbox to select.

  5. Determine if you want to use UniversalUniversal or Group-basedGroup-based auto-registration and click your selection.

    Universal will auto-register new users to a specific role of your choosing. Universal is very straight-forward: if the user attempting to login doesn't yet exist in ChangeGear, they are added automatically and assigned to the role you specify.

  6. If you select Universal, go to Step 7,

    - OR -

    If you select Group-based, go to Step 8.

  7. Click the Role drop-down and select the default role that you want ChangeGear to assign to newly added users. Go to Step 10.

  8. To configure Group-based auto-registration, use the domain utility to create the groups.

  9. Using the Windows domain group administration utility, add either the users or groups of users that should be granted access for each corresponding role.

  10. If you would like auto-registered users to consume a named license for the ChangeGear application, check the Grant auto-registered users a named license box.

    Note: If your users primarily use the ChangeGear Web application in the requester role, leave this box unchecked.

  11. Click Save to keep your configuration changes.
