Using the SmartResponder

If you have installed the intelligence pack, ChangeGear can automatically provide possible resolutions to users when they submit new Incident tickets. The SmartResponder feature leverages machine learning to help users find solutions to incidents without them having to wait for their ticket to be reviewed or assigned. When a user submits a ticket, the SmartResponder sends relevant Knowledge Base articles to the user as well as an estimated resolution time based on the resolution time of similar tickets.

ChangeGear includes a new SmartResponder template, which you can use in place of the module's default template. You can also use the SmartResponder by adding two fields – $(SmartResponder_EstimatedCompleteTime) and $(SmartResponder_SuggestedKBs) – to existing templates.

Note: The SmartResponder template and fields are only available for the Incident module.

To use the SmartResponder in a template:

  1. In the Administration tree, select Communication Templates in the Incident Management node.

  2. Select the Submit template and click Edit. The Communication Template editor appears.

  3. In the body of the template, add the following fields:

    • $(SmartResponder_EstimatedCompleteTime) calculates the estimated completion time for the ticket using the resolution time of similar tickets.

    • $(SmartResponder_SuggestedKBs) displays the two most relevant Knowledge Base articles as suggested by the machine learning service.

  4. Optionally, click Test Send to send a test e-mail so you can verify the style and format of the template.

  5. Click OK to save your modifications and close the Communication Template editor.
