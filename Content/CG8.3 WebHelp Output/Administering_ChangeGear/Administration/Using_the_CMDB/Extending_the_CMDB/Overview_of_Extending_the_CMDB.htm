Extending the CMDB

You can extend the basic configuration information in the CMDB by using the Entity Editor in the Administration Module > Advanced section. ChangeGear allows you to edit the underlying entity model and the forms associated with the entities to further match your business needs. For example, you can create CI types that ChangeGear does not provide and create new fields and forms to be used with the CI type.

In the Entity Editor (see Using the Entity Editor for more information), the Configuration Management module is unique in that each CI type - found in the CMDB Lookup Lists section - is considered a different entity, instead of a flat list of CI types, with the ability to have different fields and forms. Each CI type inherits their fields and forms from the Base CI (see What is the Base CI? for more information). You can even use the Entity Editor to create different forms to be used with sub-resource types, such as Network: Routers.

When extending your CMDB, you should have a clear idea of what you want to accomplish. For example, if you are creating a CI type, know which fields are unique to the new CI type but are not part of the Base CI so that you can add those fields to the new CI.

Important: Create a backup of your ChangeGear database and installation folders before modifying ChangeGear with the Entity Editor.
