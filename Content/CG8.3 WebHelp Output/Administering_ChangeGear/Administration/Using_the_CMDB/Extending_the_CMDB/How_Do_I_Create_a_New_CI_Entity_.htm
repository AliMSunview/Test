How Do I Create a New CI Entity?

When you create a new CI entity, a new resource type is created which is available in the Resource Type and New CI drop-downs after you have saved your changes. You can add sub-resource types from the CMDB's Lookup Lists section in Administration (see How Do I Create Sub-Resource Types? for more information). The forms, attributes, and relationships are all inherited from the Base CI, though you can modify the form for the new CI entity and add new attributes and relationships specific to the new CI entity.

To create a new CI entity:

  1. In the Administration module, select AdvancedAdvanced.

  2. Click Entity Editor in the Customization page. The Entity EditorEntity Editor will display.

  3. Select Configuration Management > CIs.

  4. With CIs selected, click Add Entity. The New Entity dialog will display.

  5. Enter the entity name. For example, if you were creating a new CI type of Storage, you would enter "Storage" in the Name field.

    Note: Entity names cannot have any special characters.

    Tip: Add fields before you save any changes to the Entity Model. This will automatically add the fields as fields on the configuration tab of the new CI Entity's default form. See How Do I Add Fields to CI Entities and How Do I Create Forms to Be Used with Specific Resource Types? for more information.

  6. Save the changes by going to File > Save All Changes.
