What is the Base CI?

The Base CI is a special CI entity from which all other CI entities inherit their fields and forms. Any fields added or removed from the Base CI will be mirrored in all other CI entities after the entity model has been saved. For example, if you wanted to add a field called "Capacity" to all CI entities, you would add it to the Base CI. This field would then be available to other CI entities in the field list of the Form Designer. If, however, you added the field to the Computer Resource entity, "Capacity" would only be available in the field list of the Form Designer for Computer Resources. See How Do I Add Fields to CI Entities? for more information.

Base CI forms are also inherited to all CI entities. Modifications made to the default Base CI form will be mirrored in all new CI forms created after the entity model has been saved while modifications made to other CI entity forms will not be mirrored in any other form.

The active Base CI form is also the default form used when creating a new CI in the CMDB. The CMDB can be configured to use specific forms for specific resource types or CI entities. See How Do I Create Forms to be Used with Specific Resource Types? for more information.
