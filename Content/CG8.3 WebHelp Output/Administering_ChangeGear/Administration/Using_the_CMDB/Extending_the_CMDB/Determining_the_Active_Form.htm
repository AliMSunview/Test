Determining the Active Form

Only one form per form type per CI entity can be active at any given time. For example, while you can have three active staff forms for the Computer Resource CI entity, only one form can be active. The active form is used as the default form when a new CI is created that is of that entity type. The active form will be used as the default form for all resources and sub-resources of this entity. If no form is active for the CI entity, the Base CI entity's active form will be used instead.

Note: You can have only one active form per form type. The Staff type is accessible to any staff user with the correct privileges. The End User type allows end users, such as requesters, access to your customized forms.

Forms can still be used if they are not active; however, this requires you to configure your resource types to use the non-active form. This is especially useful if you create sub-resource types for which you wish to use different forms. See How Do I Create Sub-Resource Types? for more information.

To activate a form from the Entity Editor:

  1. Select the form you wish to activate in the Entity Editor.

  2. Select the Active checkbox.

  3. Click Save.

To activate a form from the Form Designer:

  1. Open the Form Designer to the form you wish to activate.

  2. In the Form Designer, click Properties.

  3. Select the Active check box.

  4. Click OK.

  5. Save the form.
