Managing Dashboards and Workspaces

Administrators can edit and publish different dashboard and workspace layouts for a persona or team. These layouts define the content and placement of widgets. Please refer to the Overview of the Workspace and Using the Dashboard sections for a basic understanding of workspace features and functions.

Dashboards and Personas

ChangeGear provides built-in dashboard layouts for each Persona. These layouts act as templates for users belonging to a persona. When a user accesses the Dashboard module for the first time, the layout associated with their persona becomes their personal dashboard. The user can then modify this template, if they have the correct privileges. The resulting changes are then stored as their personal dashboard. For more information on personas, please refer to Role vs. Persona under Using the Dashboard.

The Persona field can be edited from the ChangeGear Desktop Client under Security > Roles, using the General tab. Refer to the Creating Roles section for more details. You can have roles without a persona defined, when you do not want the role to have a default layout in the workspace or dashboard, (i.e., depending on privileges for workspace and dashboard, the role may or may not access to the workspace and dashboard modules).

Dashboards and Teams

As an administrator, you can create new dashboards for teams. Each team can have multiple dashboards, which will only be visible to members of that team and to other administrators. A dashboard can have multiple teams assigned to it as well, for maximum flexibility. Team members with the appropriate privileges can view these dashboards from the web. For more information, please see Creating and Editing Dashboard Layouts.
