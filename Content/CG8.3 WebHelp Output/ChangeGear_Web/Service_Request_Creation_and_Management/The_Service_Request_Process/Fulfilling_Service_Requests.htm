Fulfilling Service Requests

When you accept a service request, you and other assignees can create task records to provide a detailed account of your fulfillment activities. For example, you can record a description of the tasks, identify the person that performed the task, and account for its cost in time and effort. You can also update the status of each task as you take it from start to completion.

When the service request has been fulfilled, you can request validation from the requester that the service request is sufficiently fulfilled.

To create a new task:

  1. Click the service request to open.

  2. Click the Tasks tab on the lower half of the service request window.

  3. In the Tasks tab toolbar, click New.

  4. In the Task, select the Implementer from the list or click to select dynamic role assignment for Implementer and/or other types of Implementers.

  5. In the Description field, type a brief description of the new task.

  6. In the Type list, select the appropriate task type.

  7. In the Start fields, select the start date and times of the implementation task.

  8. Enter the Duration of the task.

  9. In the Complete fields, select the start date and times of the implementation task.

  10. Enter the Effort of the task.

  11. In the Notes box, type any explanations or task requirements.

  12. Click Save.

To request review for a service request:

  1. Double-click on the service request to open.

  2. In the toolbar, click Request Validation.

  3. Click OK.
