Service Request Data Fields

With few required fields, the service request form is designed to be flexible in how much information is recorded per service request. Each service request record includes a record of the requester and the initial details of the service request. The service request form is designated by request type in order to display the appropriate form (data fields) associated with a particular request.

ChangeGear automatically populates the Requester, E-Mail, Phone, Location, Department, Request Type, and the Created date and time fields. ChangeGear also automatically assigns the next sequential unique ticket ID number available for the module.

If you want to change the Requester, begin to type the name in the Requester field. ChangeGear auto-completes the rest of the name or narrows the choices in the drop-down menu. Use the button by the user fields (Requester, Owner, Assigned To) to select a team.

You can Identify Impacted Resources when the form for a particular request type contains these fields.

You cannot create a service request if you do not enter information into the required fields (marked in red).

Tip: Forms may have collapsed field data for less frequently viewed fields. You can expand fields by clicking the arrow on the form to edit/view additional fields.

For ChangeGear users that are not end users, the service request form displays a VIP icon next to the Requester if the Requester has been designated a VIP user by the ChangeGear administrator. The VIP status of a ticket is determined when the ticket requester is auto-filled or modified in a ticket. If a user is marked by the ChangeGear administrator as no longer having VIP status, any previous tickets requested by that user are still marked as VIP unless the requester is modified.
