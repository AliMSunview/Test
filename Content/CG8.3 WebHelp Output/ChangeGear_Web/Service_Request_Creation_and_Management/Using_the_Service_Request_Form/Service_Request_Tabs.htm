Service Request Tabs

The tabs at the bottom of the service request form provide more tools that you need to effectively manage your service requests.

Description

Enter the description text and/or add notes in the Description tab. The Description and Internal Notes fields both use a commentary control that captures the name of the person adding to the fields as well as the Time and Date of the new entry. If your ChangeGear administrator has enabled the Conversations feature, you can also send correspondence to users via e-mail directly from the service request form. For more information, see Using Conversations.

Once the ticket is saved, the entries are listed chronologically and are read only. The Description field is viewable by End Users. The Notes field also records when and who updates the field and can only be viewed by other staff members.

Tasks

Create tasks to be completed by other staff members or add business tracking information like start/end date and time. The Tasks tab cannot be seen by End Users.

Approval

Add the appropriate personnel to the Approver list to validate the request. A ticket can have multiple Approvers. See Approving Service Request for more information.

Related Items

Search for related items—such as problems and other tickets, Knowledge Base (KB) articles—and link to them. You can unlink items that do not apply. See Linking to Related Items for more information.

Attachments

Add or view attachments that help to resolve the service request. See Attaching Items for more information.

SLA

If the ChangeGear administrator has enabled SLA, the SLA tab displays in the service request form. This tab provides all of the SLA information for this service request ticket.

History

ChangeGear automatically keeps a history of the changes made to a service request, including who made changes to the ticket and what the changes were. You can add history items, such as when you last tried to contact a requester to the service request report. See Managing History for more information.
