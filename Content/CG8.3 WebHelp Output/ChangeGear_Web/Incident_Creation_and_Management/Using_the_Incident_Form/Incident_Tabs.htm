Incident Tabs

The tabs at the bottom of the incident form provide more tools that you need to effectively manage your incidents.

Discussion

Enter any communications and/or notes in the Discussion tab. The Conversation History and Internal Notes fields both use a commentary control that captures the name of the person adding to the fields, as well as the time and date of the new entry. If your ChangeGear administrator has enabled the Conversations feature, you can also send correspondence to users via e-mail directly from the incident form. For more information, see Using Conversations. Once the ticket is saved the entries are listed chronologically and are read only.

Note: The Conversation History field can be seen by End Users. To restrict correspondence to staff members, use the Internal Notes field.

User Sentiment

This tab also lets you display a chart of user sentiment over the lifetime of the ticket. Sentiment reflects the end user's satisfaction regarding the status of the ticket and is automatically calculated for each comment that the end user makes. Comments that appear to be generally positive result in positive sentiment, negative comments result in negative sentiment, and neutral comments result in no change in sentiment. This gives staff users insight into the end user's experience with the ticket without having to first read through the entire conversation. For more information on sentiment, see Viewing User Sentiment under Managing Incidents.

Note: You may not have access to this feature depending on your ChangeGear configuration. Sentiment cannot be viewed by end users.

To view a chart of user sentiment, click the sentiment icon in the Conversation History field. This icon only appears when an end user adds a comment to the conversation. Clicking the icon opens the Sentiment Chart where you can see the sentiment calculated for each comment in the Conversation History. Hovering the mouse cursor over a point in the chart shows you the sentiment value for that particular comment, as well as highlights the corresponding comment in the Conversation History. Click the icon again to close the graph.

Tasks

Create tasks to be completed by other staff members or add business tracking information like start/end date and time. The Tasks tab cannot be seen by End Users.

Related Items

Search for related items—such as problems, other incidents, and KB articles—and link to them. You can unlink items that do not apply. See Linking to Related Items for more information.

Resolution

Enter the resolution to the incident in the Resolution field. Enter how long you worked on the incident, how much it cost, and the close type.

Attachments

Add or view attachments that help to resolve the incident. See Attaching Items for more information.

SLA

If the ChangeGear administrator has enabled SLA then the SLA tab displays in the Incident form. This tab provides all of the SLA information for this Incident ticket.

History

ChangeGear automatically keeps a history of the changes made to an incident, including who made changes to the ticket and what the changes were. You can add history items, such as when you last tried to contact a requester to the incident report. See Managing History for more information.
