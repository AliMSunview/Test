Viewing User Sentiment

Sentiment describes the overall satisfaction that an end user–in most cases, the ticket requester–feels towards the ticket. You can quickly determine whether a user is satisfied about the state of a ticket without having to open the ticket and read through the conversation history.

Note: This feature may not be available depending on your ChangeGear configuration.

Sentiment works by analyzing the words used in an end user's comment. If the words are generally positive, you can infer that the user is satisfied about the direction that the ticket is taking. If the comment seems negative, you may want to reach out to the user or dedicate more resources to the ticket. A comment might also have no strong positive or negative connotations, in which case it has a sentiment of zero.

You can view user sentiment in two locations in the Incident module: on the grid and on the Incident form. For more information on viewing sentiment on the Incident form, see the User Sentiment section of the Incident Tabs topic under Using the Incident Form.

Using the Ticket Grid

The Incident module ticket grid provides two columns for viewing user sentiment:

  • Sentiment displays the end user's current satisfaction level based on their most recent comment. If the comment was positive, the column displays a happy face. If the comment was negative, the column displays a frowning face. If the comment was neutral, there are no end-user submitted comments in the ticket, or if sentiment could not be determined, then the column displays a neutral face.

  • Sentiment Trend displays the most recent change in sentiment by comparing the end user's latest comment to the comment just before it. If there was an increase in sentiment, the column shows an up arrow, otherwise it shows a down arrow. It is possible for a ticket to show a positive sentiment with a negative sentiment trend, as well as the other way around.

These columns can be added to any view. They can also be found in the global Smart View view. For more information on adding columns to a view, please see Using the Grid > Adding and Removing Columns.
