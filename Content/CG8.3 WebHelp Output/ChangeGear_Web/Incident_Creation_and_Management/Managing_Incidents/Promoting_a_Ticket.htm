Promoting a Ticket

If a ticket has been identified as a problem, (i.e., the problem affects more than one user with similar symptoms) or the ticket can only be resolved with a change to resources in your organization, you can promote the ticket to a problem or change request. If you know the incident, problem, change request already exists and that other technicians are working on a resolution, you should link to the incident, service request, problem, or change request via the Related Items tab.

See Promotion in Leveraging ChangeGear Integration Features for a complete explanation of the workflow states.

Note: You can promote service requests to any ticket module, (e.g., incidents, problems, and change requests). However, no modules can promote to service request due to the multiple form types available for service requests, (e.g., Access request, Move request, etc.).

To promote a ticket:

  1. Open the affected ticket.

  2. Click Promote on the toolbar. The Select Items dialog displays.

  3. Search for an existing incident, problem, or change request by entering a search term in the search field, selecting an appropriate tab (e.g., Problems, RFCs), and clicking the Find button.

  4. If a problem, incident or change request already exists in the system, select the item and click Promote to a Selected Item,

    - OR -

    If a ticket does not exist in the system, click on a particular button Promote to a new..... to create a new ticket in another ticket module.

  5. A confirmation dialog displays. You can view the ticket using the link on the confirmation dialog.

  6. Click OK to return to the ticket.

After a ticket has been promoted, the associated ticket appears in the Related Items tab of the service request.

Promoting to a New Service Request

Promoting a ticket to a new Service Request ticket requires additional configuration changes. Instead of offering a default form, the Service Request module requires users to select one of several forms based on the type of ticket. Since the Promote to Service Request action creates a new Service Request ticket, the form to use must be specified in the promote template. This is accomplished by specifying the EditorLayout field to the desired form layout name in the in the server installation directory's EntityTemplates.xml file.

To set the Service Request promotion form:

  1. As an administrator, open the EntityTemplates.xml file in the ChangeGear server installation directory (defaults to C:\Program Files(x86)\ChangeGear\Server\EntityTemplates\EntityTemplates.xml).

  2. Scroll to or search for the EditorLayout field and set its value to the name of the form that you wish to use. To use one of these forms, enter its name without spaces. For example, General Request becomes GeneralRequest.

    Tip: You can also specify a variable name (such as a module field). For example, you can create a user-defined lookup list containing all available Service Request layouts and add it to the Incident form, allowing users to select a Service Request type themselves.

  3. Save the file, then restart the ChangeGear server.
