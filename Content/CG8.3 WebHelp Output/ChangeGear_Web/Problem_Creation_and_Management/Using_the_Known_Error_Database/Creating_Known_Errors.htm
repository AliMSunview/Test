Creating Known Errors

A known error is a problem that has been identified, but your organization has either made no plans to rectify it or long-term plans have been made that will not solve the problem in the short term. It will generally have a workaround and the probable root cause identified. Problems may be set as known errors if an upcoming hardware or software upgrade will fix the problem or if the fix requires a complicated procedure and few users will see the problem. Workarounds for known errors are often worded such that help desk staff members walk the end user through the steps.

Note: You must have appropriate privileges to edit known errors.

For example, the help desk may receive multiple incidents about not being able to log in to the training application. An incident, linked to the other incidents, is promoted to a problem. Your IT team looks into the problem and discovers that the training software has a memory leak causing the server to restrict access to only users already logged in. Your organization notifies the training software vendor of the problem, and promises to resolve the issue in the next release. The temporary fix is to install more memory on the server, and a change ticket is created. The temporary workaround is for users to wait ten minutes and try again. The problem is marked as a known error, as it may take some time for the solution to be implemented.

To create a known error:

  1. Open the Problem module

  2. Open the problem ticket identified as a Known Error.

  3. In the actions toolbar, click Known Error.

Note: You must have known error privileges to view the action.

The problem ticket will now be available in the Known Errors view of the Problem module and all throughout the product. See Viewing Known Errors for more information.
