Problem Data Fields

The problem data fields are the main information entry points for the problem report. With few required fields, the problem form is designed to be flexible in how much information is recorded per report. Each problem report includes a record of the requester and the initial details of the problem.

When you open a new problem, ChangeGear automatically populates the Requester, Requester E-mail, and Created fields. If you want to change the Requester, begin to type the name in the Requester field. ChangeGear fills out the rest of the name or narrow down the choices in the drop-down menu.

You can add Impacted Resources or Users by clicking on the button by the field, clicking the Add button, and then adding resources or users to the Selected Items section. Users can also include teams or contacts. If you have a drop-down menu option on the Impacted Resources field, see Identifying Impacted Resources for more information.

You may also select teams for the Owner and Assigned To fields by clicking on the button by the field.

You cannot create a problem report if you do not enter information into the following required fields (marked in red): Requester, Summary, Due Date, Problem Type, Impact, Urgency, and Priority.

You can add a more detailed summary of the item in the Description field. If your ChangeGear administrator has enabled the Conversations feature, you can also send correspondence to users via e-mail directly from the problem form. For more information, see Using Conversations. Enter the steps taken to resolve the problem in the Resolution field.

For ChangeGear users that are not end users, the Problem form displays a icon next to the Requester if the Requester has been designated a VIP user by the ChangeGear administrator. The VIP status of a ticket is determined when the ticket requester is auto-filled or modified in a ticket. If a user is marked by the ChangeGear administrator as no longer having VIP status, any previous tickets requested by that user are still marked as VIP unless the requester is modified.

Tip: Forms may have collapsed field data for less frequently viewed fields. You can expand fields by clicking the arrow on the form to edit/view additional fields.
