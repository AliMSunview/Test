Defining Role Privileges using Web Admin

You can grant or deny privileges to your users from the Roles module in theWeb Administration section of ChangeGear. Privileges are managed on a per-workflow basis. If a module has multiple workflows enabled, you can select each workflow individually from the drop-down. Selecting the name of the module lets you set privileges for the module's default workflow.

Note: Selecting the module name exposes additional privileges, such as Access Module and View. Disabling these privileges may prevent users from accessing the module in the ChangeGear Web client.

To define Role privileges:

  1. From the Web Administration page, click Security> Roles. ChangeGear displays the Roles page.

  1. Hover over role, then click the pencil icon. ChangeGear opens the slide-out Role editor. Click the Privileges tab.

  1. In Modules, click the name of the module for which you want to define privileges.

    WARNING: Always review a workflow's privileges before making it the default workflow.

  2. In Privileges, click the check box to enable each privilege you want to enable or disable for the role.

  3. In Workflow, (for modules with enabled workflows) hover over the workflow and click the trashcan icon to delete it from this role.

  4. Click Save  to save your changes.
