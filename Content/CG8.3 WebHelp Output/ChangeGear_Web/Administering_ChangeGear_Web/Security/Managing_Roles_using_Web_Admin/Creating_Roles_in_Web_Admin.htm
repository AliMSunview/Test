Creating Roles in Web Admin

ChangeGear comes with a predefined set of service desk roles. You can, however, modify the descriptions and privileges of these roles to meet your organizational requirements. You can also add new roles if you don't find anything that applies to your organization’s roles.

Tip: The roles listed in the Web Administration module populate the Roles field in the Users Details page.

Note: You cannot assign Site License users (End Users or Requesters) to a custom role.

To create a Role:

  1. From the Web Administration page, click Security> Roles. ChangeGear displays the Roles page.

           

  1. To add a new role, click the plus sign in the upper right corner. ChangeGear opens the slide-out Role editor.

  1. Enter a Name for this role. In the Description field, type a brief description of the role.

  2. The Persona field directly relates to defined layouts for dashboard and workspace when the role has access privileges to these functions. When persona is blank, the system defaults the dashboard and workspace layouts to the IT Staff published layout. In addition to accessing the role's persona layout, the user can also personalize My Workspace or My Dashboard layouts to suit their needs.

    Refer to the User's Guide, Using the Dashboard and Using the Workspace sections for a complete explanation of layouts (personas).

  3. You can edit role membership, define role privileges, or define filed security define role privileges, or define field security at this time.

  4. Click the save icon to create the new role.
