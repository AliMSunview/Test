Defining Field Security using Web Admin

You can restrict users' access to certain fields on a form by applying field security. Requesters, or end users, already view restricted versions of staff user forms, but you can fine-tune staff user forms by setting the visibility of fields for each role. You can also control which fields are read-only, meaning they can't be modified by users. Field security is available for the Change, Incident, Problem, and Service Request modules.

For example, if you do not want IT Staff users to see the Rollback Plan then you can hide the field, but if you want IT Staff users to view the Origin ID without being able to edit it then you can make that field read-only.

Important: Make sure that you do not mark a required field as hidden or read-only. Users may experience errors if they try to save or submit a ticket without access to required fields.

To define field security:

  1. From the Web Administration page, click Security> Roles. ChangeGear displays the Roles page.

  2. Hover over role, then click the pencil icon. ChangeGear opens the slide-out Role editor. Click the Privileges tab, then click the Fields tab.

  1. Select the Module for which you would like to restrict field access.

  2. Select the Workflow for which you would like to restrict field access.

    Note: You can only select enabled workflows from the drop-down. Leave this field blank to modify the default workflow.

  3. Select the ticket State for which you would like to restrict field access. Your choices are New, Pending Close, Unsubmitted, In-Progress, and Pending Resolution.

  4. Click the System tab to see information ChangeGear collects about this role, including OID, Created by, Modified by, Modified (time/date), and Accessed by.

  5. Click Save  to save your changes.
