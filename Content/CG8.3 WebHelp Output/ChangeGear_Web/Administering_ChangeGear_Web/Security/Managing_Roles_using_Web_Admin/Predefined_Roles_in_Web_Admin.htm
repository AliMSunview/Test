Predefined Roles in Web Admin

ChangeGear comes with a predefined set of change management roles. You can, however, modify the descriptions and privileges of these roles to meet your organizational requirements. You can also add new roles if you do not find anything similar to your organization’s roles in ChangeGear.

The roles listed in the table below populate the Role drop-down list in the User slide-out editor.

Role

Responsibility

Administrator

Configures and maintains the change management system.

Anonymous

Views Announcements.

Change Advisory Board (CAB)

Submits change requests.

Change Authority

The Change Management team.

Change Manager

Oversees change requests throughout the change management life cycle.

Change Requesters

Submits change requests.

Emergency Committee

A sub-set of the full Change Advisory Board who make decisions about Emergency Changes.

Executive

The executive role can use the ChangeGear dashboard, knowledge base and create tickets, (e.g., incident, problem or change tickets).

Help Desk

Works at the help desk and serves on the front lines with customers.

Note: IT staff members in this role require notification regarding any change that might impact customers.

Incident Manager

Oversees incident requests throughout the incident management life cycle, including knowledge base management.

Incident Requesters

Submits incident requests and service requests.

Note: There is no separate role for Service Requesters. Incident Requesters have the same role in the service request module.

Incident Team member

Responsible for all aspects of Incident management.

IT Staff

Receives change request assignments for implementation.

Problem Manager

Oversees problem requests throughout the problem management life cycle, including knowledge base management.

Problem Requesters

Submits problem requests.

Problem Team member

Responsible for all aspects of Problem management.

Release Manager

Responsible for all aspects of release management process.

Reviewer

Reviews change requests to ensure their relevancy and their readiness for the approval process.

Service Catalog Manager

Responsible for administration of the Service Catalog.

Ticket Requesters

Users that are requesters for Change, Problem, Incident and Flex modules.
Security Team Requesters Allows requesters (end users) the ability to view and edit tickets their team owns. Requesters can be added to this role when setting up Security Teams.
