Dynamic Roles in Web Admin

When working with approval group details, business rules, and notification rules, you can edit and program business rules that trigger specific actions. These actions help ensure timely completion and communication of requested changes and relieve management and staff of many manual tasks such as:

  • Electronic assignments or escalation of requests

  • Automatic notification of important developments

  • Follow-up reminders

  • Third-party application monitoring tools

You may need these actions to refer to a role, such as an Approver, an Assignee, or a QA Owner and not a specific user. ChangeGear lets you select Dynamic Roles which dynamically represent a role instead of an individual user.  ???

For example, you want to notify all of the users on the current RFCs Approver list about your change request. By selecting the Approvers dynamic role, ChangeGear dynamically represents all of these users.
