Security Teams using Web Admin

ChangeGear allows you to create security teams that can include members from different roles, locations, or other organizational divisions. Members of security teams can be members of notification teams or other security teams. Each team member has a separate role within the team that takes precedence when a ticket or item is owned by a security team.

Note: Site-licensed users can only be assigned to Requester roles within a security team.

For example, George is part of the IT Staff, and wants to know the status of a ticket owned by the Database Security Team. The Guest Role of the Database Security Team is set to No Privileges, and George is not part of the Database Security Team. George cannot view the ticket, and will have to ask a member of the security team about the status.

George, however, is part of the Systems Team, a security team, as a Change Requester. He can perform anything that a Change Requester has the privileges to perform on tickets or items owned by the Systems Team.

If the Systems Team assigns a ticket or item they own to the Database Security Team, the Database Security Team will have the Guest Role of Help Desk.

Creating or using teams is not a requirement to use ChangeGear; however, security teams expand security and usability by restricting tickets and configuration items (CIs) to those that need to view the tickets/items.

The role of a user per owned item is determined by the member's team role. For more information, see Determining User Privileges.

Note: New or modified team members must refresh their ChangeGear client or re-login to ChangeGear to view changes to their roles within the Team.

To add a Notification Team, see Adding Teams Using Web Admin.
