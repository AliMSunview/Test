Notification Teams with Web Admin

ChangeGear allows you to create notification teams that can include members from different roles, locations, or other organizational divisions. Notification teams are similar to a group e-mail or alias in your directory as you do not need to select every user every time. For example, you can create a Troubleshooting Notification Team that includes IT members from different organizational units across your company. You can assign tickets to this team and have all members of the team notified of the assignment. Members of notification teams can be members of other notification teams or be part of security teams.

You may also include Active Directory (AD) groups in your notification teams. Members of these external groups are notified along with your ChangeGear members. However, the users accounts in the groups must be enabled in AD and must have an associated e-mail address to receive notifications.

Creating or using teams is not a requirement in ChangeGear, however, teams expand usability by allowing you to quickly communicate with multiple users on the same team.

Important: Notification team members must have the Team View privilege enabled to view tickets assigned to the team. However, if the ticket or Configuration Item (CI) is owned by a security team, the security team privileges apply.

Note: New or modified team members must refresh their ChangeGear client or re-login to ChangeGear to view changes to their roles within the team.

To add a Notification Team, see Adding Teams Using Web Admin.
