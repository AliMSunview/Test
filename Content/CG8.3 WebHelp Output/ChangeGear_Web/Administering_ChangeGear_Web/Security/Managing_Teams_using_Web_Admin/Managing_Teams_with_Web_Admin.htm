Managing Teams with Web Admin

ChangeGear has two types of teams: notification team and security team. When you create a team, you can designate the type of the team.

  • A notification team receives e-mail notifications or assignments to groups of users, (i.e., team), with potentially disparate ChangeGear roles.

  • A security team provides additional security mechanisms when used in conjunction with the Owner field. A team designated as a security team allows users within the same team to assume different roles.

In ChangeGear Web Administration, you can navigate to the Team Manager page by clicking Security>Teams. Here you define teams which receive communications, (such as e-mail notifications), and assign roles to members of security teams. In addition, you can define team owners and team e-mail addresses.

You can add/remove team members using Team Manager via ChangeGear on the web.

Note: You may not have access to this ChangeGear feature depending on your permissions or ChangeGear edition.

The following example describes how a person can be part of a notification team and/or a security team and the implications.

Ted is the help desk manager with a role of Incident Manager. Because Ted needs to know of upcoming changes, he is part of the IT Management Notification Team, which notifies team members of all upcoming system changes and impact.

Ted is also monitoring the e-mail server upgrade and has been granted limited access to the System Administration Team, a security team, with a role that allows him to view e-mail upgrade associated tickets. Non-team members have no privileges or access to view tickets or configuration items (CIs) owned by the Systems Team.
