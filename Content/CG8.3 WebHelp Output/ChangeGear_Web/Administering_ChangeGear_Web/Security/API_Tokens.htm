API Tokens

An API token is a string that is provided when making API calls and is sent as part of the HTTP request header. A 401 Unauthorized or invalid token response is returned for invalid API tokens.

The main purpose of the API token is for use with ChangeGear REST APIs which makes authentication and hence integration simpler without the need for user credentials.

The API token(s) are associated with a user account. The creation of token(s) is limited to users who have “Administer Security” privilege. Users with this privilege can create a token for their own user account or for a service account. The API token will use the role privileges and permissions associated with its user account when performing operations.

For security reasons, once a token is generated, it cannot be updated or viewed. It needs to be kept safe like any other secure information like passwords. An API token(s) does not expire. An API token can be revoked as needed. Once revoked, the token cannot be used anymore and any existing API calls using the token will fail.

To create an API token:

  1. Log in to the web application and navigate to the web administration page.
  2. Navigate to Security->Users and double-click on your user account or on a service account for which you want to create an API token.
  3. Click the API Tokens tab.
  4. Click on the + button as seen below:

  1. Enter the name of the token and click on create. Copy the token and keep it in a safe place.

  1. Close the slide-out and return to the API token grid. The Last Accessed time of the token is updated any time an API call is made with the token.

  1. An API token cannot be updated and can be revoked by hovering over the row and clicking on the Revoke action button.

  1. The maximum number of API tokens that can be created is 25.

Using an API Token

The API token must be sent in as one of the request headers with key name as “apitoken” and the value as the token string.
