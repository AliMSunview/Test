Inbound and Outbound Communications

Before setting up inbound and outbound communication on a connection, webhooks/bots need to be configured on the application you are integrating with. For inbound communication, the application will provide either the Hash-based Message Authentication Code (HMAC) or the Security token needed in ChangeGear. For outbound communication, the application will provide the URL needed.

  1. Slack documentation available at https://api.slack.com/
  2. Microsoft Teams documentation available at https://docs.microsoft.com/en-us/microsoftteams/platform/webhooks-and-connectors/what-are-webhooks-and-connectors

Notes: During this part of the integration you need to enter a Callback URL for both Slack and Teams.  The URL refers to the system on which the integration is being set up, and looks something like the following example:

           - appname will always be “msteams” for integration with MS Teams, and “slack” for integration with Slack.
           - entitytype is the capital letter I (as in Indiana), followed by the ChangeGear module being associated (e.g., Incident)
           - connection is the connection you previously named

Setting up Inbound Communication

1.  Go to the Inbound tab in the Communication area and provide the corresponding inbound security token/ (HMAC for Microsoft Teams, or Signing Secret for Slack).

Note: Once you save the token, it becomes masked (*****) and is no longer visible.

2.  Make sure to Enable the connection.

Setting Up Outbound Communication

1.  Go to the Outbound tab in the Communication area and provide the corresponding outbound URL.

2.  Save () the connection.

Configure ChangeGear Desktop Client for Outbound Communications
‎(Required for both Microsoft Teams & Slack)

Setting up the outbound communication in the Desktop Client allows ChangeGear to alert the external application channel when a ticket has been created or updated.

The configurations required in the Desktop Client are made in the Workflow. .

1.  Log into ChangeGear Desktop Client.

2.  In Administration, under the node that corresponds to the module you configured (e.g., Incident Management), click the Workflow option to display existing workflows (including the incoming integration you just created in the previous sections).

3.  Double-click on the workflow that corresponds to the outbound communication you want to setup.

4.  Select the action (submit, etc.) on which you want to setup the trigger for an outbound. Under the Actions tab, click on the Automation tab.

5.  Click the Add icon () to display the Create an Action prompt.

6.  In the Action Type drop-down, select Run Integration, and click OK to display the Run Integration Details window.

7.  In the Run Integration Details window, add a description, and make the following selections:

  • Integration – select the external app being used (i.e., Microsoft Teams or Slack).
  • Project – select the connection name.
  • Field Mapping – select the Mapping set you want to use.

8.  Click OK to complete the settings and close the Run Integration Details window. The new action appears in the list. Go to File>Save As to save the settings and return to the workflow setup. The new workflow appears in the list.

IMPORTANT! Be sure appropriate Role Privileges are set before going to the next step.

9.  In the Common Tasks list, select Enable and Set as Default to make the workflow available.

10. Save () the workflow

Once you have completed the steps shown above, you need to create Mapping, then set up Authentication.
