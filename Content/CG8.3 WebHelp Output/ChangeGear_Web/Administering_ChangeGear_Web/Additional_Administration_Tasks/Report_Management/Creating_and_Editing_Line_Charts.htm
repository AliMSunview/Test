Creating and Editing Line Charts

In the Report Designer, you can create line charts which statistically compare data within a specific Change Gear module. The data is reported as one or more lines. You can define which module columns are represented in the line chart's X Axis and plotted lines, as well as apply filtering to further refine the scope of data represented.

To create a line chart report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Click the New icon in the reports tool bar.

  3. The report creation dialog displays. Select the line chart report.

  4. Select a module type for the table data used to create the report, (e.g., incident, problem, CMDB, assets, etc.). For example, select the Incident module to create a report regarding Incident tickets.

  5. Select a view for the report, (e.g., All Incidents, Active Incidents, Closed Incidents, etc.). When a line chart is based upon a view, the report uses the filter criteria of that view.

  6. Click Next. The line chart creation screen displays.

  7. Enter the Name of the report.

  8. Optionally, you can enter a Description for the report.

  9. Select Report Category from the list. The Report Category determines which category within the module to save the report. The My Reports category saves the report in your own My Reports and is not visible to other users.

Data Tab

The Data Tab allows you to define data in a particular module to display on the line chart.

  1. The X Axis field allows you to specify which module table column displays as the line chart's X Axis, (the horizontal line at the bottom of the line chart).

    For example, suppose you want to show the total number of incident tickets per location. You would set the X Axis field to Location Name. The generated line chart renders each unique location name across the X Axis and each unique location has a data point plotted on the chart showing the number of incident tickets in that location. A line is drawn between the location incident numbers plotted to show the trend of incidents by location.

  2. Optionally, you can specify a module table column as Series 1. When Series 1 is specified, each X Axis data point represents a unique value for Series 1 in that X Axis data point. For example, suppose you want to show the total number of incident tickets per location but also further group the tickets by incident Status. You would set the X Axis field to Location Name and set the Series 1 field to Status. The generated line chart renders each unique location name across the X Axis and each unique location has one or more vertical dots representing the count of tickets with a specific Status level in that location.

  3. Optionally, you can also enable a second series to the line chart Series 2. Series 2 automatically uses the same module table column as Series 1 and is useful when you use different filtering criteria for each of the series.

  4. For example, suppose you want to show the total number of incident tickets per location and also group the tickets by New vs. In-progress status. You would set the X Axis field to Location Name, set the Series 1 field to Status and enable Series 2. Using the criteria filtering, (see below), you would set the Series 1 filter to only show the New tickets and Series 2 filter to show the In-progress tickets. The generated line chart renders each unique location name across the X Axis and each unique location has data point lines plotted as two different colors, one for each series, (new and in-progress).

Criteria Filter Tab

The Criteria Filter tab allows you to add filtering criteria to the line chart. Please refer to the Filtering Columns Table Report section in Creating and Editing Table Reports.

X Axis

This basic filtering allows you to further refine which tickets/items to include in the line chart other than the column equal value criteria in the Data Tab. You can optionally apply filtering to the X Axis. This filtering allows you to filter which values of the X Axis column are displayed on the line chart report's horizontal axis.

Series 1

You can optionally apply filtering to the Series 1 column. This filtering allows you to filter which values of the Series 1 column are used to calculate the Series 1 sum total, (i.e., the height of the Series 1 data point).

Series 2

You can optionally apply filtering to the Series 2 column. This filtering allows you to filter which values of the Series 2 column are used to calculate the Series 2 sum total, (i.e., the height of the Series 2 data point). 

Create Report

The report is rendered after saving the report. To create the report, click Create.

Note: You can create new reports from existing reports by using the Save As option.

Editing a Line Chart Report

ChangeGear allows you to edit a line chart report and modify the filter criteria and other criteria similar to creating a table report.

To edit a line chart report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Select a line chart report from the Reports Module grid.

  3. Click the Edit icon from the line chart tool bar.

  4. You can edit the Description as necessary.

  5. The system allows you to change any report Data Tab and Criteria Filter Tab as shown above in the Create a Line Chart Report section.

  6. Click Save.

    Note: You can create new reports from customized reports by using the Save As option.

Predictive Line Charts

In addition to standard line charts, ChangeGear includes predictive line charts. Predictive line charts analyze ticket data to predict future trends such as the number of tickets that will be opened, closed, or assigned on certain days. Predictive charts cannot be created or modified through the ChangeGear Web client. For more information on how to use predictive line charts, please see Incident Creation and Management > Managing Incidents > Predicting Trends.
