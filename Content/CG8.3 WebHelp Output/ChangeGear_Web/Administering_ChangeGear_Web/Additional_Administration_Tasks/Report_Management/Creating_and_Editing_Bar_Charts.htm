Creating and Editing Bar Charts

In the Report Designer, you can create bar charts which are diagrams with vertical bars that statistically compare data within a specified ChangeGear module. You can define which module columns are represented in the bar chart's X Axis and series as well as apply additional filtering to further refine the scope of data represented.

To create a bar chart report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Click the New icon in the reports tool bar.

  3. The report creation dialog displays. Select the Bar Chart report.

  4. Select a module type for the table data used to create the report, (e.g., incident, problem, CMDB, assets, etc.). For example, select the Incident module to create a report regarding Incident tickets.

  5. Select a view for the report, (e.g., All Incidents, Active Incidents, Closed Incidents, etc.). When a bar chart is based upon a view, that report uses the filter criteria of that view.

  6. Click Next. The Bar Chart creation screen displays.

  7. Enter the Name of the report.

  8. Optionally, you can enter a Description for the report.

  9. Select Report Category from the list. The Report Category determines which category within the module to save the report. The My Reports category saves the report in your own My Reports and is not visible to other users. You could use My Reports to work on a report before saving it into one of the module categories so others can access the report.

Data Tab

The Data Tab allows you to define data in a particular module to display on the bar chart.

  1. The X Axis field allows you to specify which module table column to represent as the bar chart's X Axis (the horizontal line at the bottom of the bar chart).

    For example, suppose you want to show the total number of incident tickets per location. You would set the X Axis field to Location Name. The generated bar chart renders each unique location name across the X Axis and each unique location has a single vertical bar showing the total number of incident tickets in that location.

  2. Optionally, you can specify a module table column as Series 1. When Series 1 is specified, each X Axis data point renders one or more vertical bars each representing a unique value for Series 1 in that X Axis data point.

    For example, suppose you want to show the total number of incident tickets per location but also further group the tickets by incident Priority. You would set the X Axis field to Location Name and set the Series 1 field to Priority. The generated bar chart renders each unique location name across the X Axis and each unique location has one or more vertical bars representing the count of tickets with a specific Priority level in that location.

  3. Optionally, you can also enable a second series to the bar chart Series 2. Series 2 automatically uses the same module table column as Series 1 and is useful when you use different filtering criteria for each of the series.

    For example, suppose you want to show the total number of incident tickets per location and also group the tickets by High Severity vs. Low Severity. You would set the X Axis field to Location Name, set the Series 1 field to Priority and enable Series 2. Using the criteria filtering, (see below), you would set the Series 1 filter to only show the highest priority levels and Series 2 filter to show the lowest priority levels. The generated bar chart renders each unique location name across the X Axis and each unique location has two vertical bars, one for each series, (high severity and low severity).

Criteria Filter Tab

The Criteria Filter tab allows you to add filtering criteria to the bar chart. Please refer to the Filtering Columns Table Report section in Creating and Editing Table Reports.

X Axis

You can optionally apply filtering to the X Axis. This filtering allows you to filter which values of the X Axis column are displayed on the bar chart report's horizontal axis.

Series 1

You can optionally apply filtering to the Series 1 column. This filtering allows you to filter which values of the Series 1 column are used to calculate the Series 1 sum total, (i.e., the height of the Series 1 vertical bar).

Series 2

You can optionally apply filtering to the Series 2 column. This filtering allows you to filter which values of the Series 2 column are used to calculate the Series 2 sum total, (i.e., the height of the Series 2 vertical bar). 

Create Report

The report is rendered after saving the report. To create the report, click Create.

Note: You can create new reports from existing reports by using the Save As option.

Editing a Bar Chart Report

ChangeGear allows you to edit a bar chart report and modify the filter criteria and other criteria similar to creating a bar chart report.

To edit a bar chart report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Select a bar chart report from the Reports Module grid. Click the Edit icon from the bar chart tool bar.

  3. You can edit the Description as necessary.

  4. The Report Designer allows you to change any report Criteria Filter Tab or Data Items as shown above in the Create a Bar Chart section.

  5. Click Save.

    Note: You can create new reports from existing reports by using the Save As option.
