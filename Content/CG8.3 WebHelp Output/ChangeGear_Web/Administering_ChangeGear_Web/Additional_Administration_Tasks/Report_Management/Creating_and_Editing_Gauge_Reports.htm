Creating and Editing Gauge Reports

In the Report Designer, you can create Gauges which perform a calculation for a subset of tickets/items in a ChangeGear module. The calculation is shown as a percentage (0% to 100%) of all tickets.

To create a gauge report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Click the New icon in the reports tool bar.

  3. The Report Creation dialog displays. Select the Gauge report.

  4. Select a module type for the table data used to create the report, (e.g., incident, problem, CMDB, assets, etc.). For example, select the Incident module to create a report regarding Incident tickets.

  5. Select a view for the report, (e.g., All Incidents, Active Incidents, Closed Incidents, etc.). When a Gauge is based upon a view, that report uses the same filter criteria as the view.

  6. Click Next. The Gauge Creation screen displays.

  7. Enter the Name of the report.

  8. Optionally, you can enter a Description for the report.

  9. Select Report Category from the list. The Report Category determines which category within the module to save the report. The My Reports category saves the report in your own My Reports and is not visible to other users. You could use My Reports to work on a report before saving it into one of the module categories so others can access the report.

Data Tab

The Data tab allows you to define the data for the gauge chart.

The Column field identifies which module table column to compare in the gauge chart.

The Column Value field enables you to specify the value used to report the percent on the gauge as compared with the overall ticket total percent.

The Operation field has the count and sum options. You can count the number of tickets which shows as a percentage on the gauge or you can sum up all the values for the column field such as Cost.

Note: The sum option does not work for non-numeric fields like Assigned to or Due date.

For example, suppose you want to show the percentage of the total incident tickets In-Progress on a gauge chart. You would set the Column field to Status and the Column Value to In-Progress. The generated gauge renders a percentage of the number of tickets that are currently In-Progress as the gauge dial number in comparison with the total number of tickets, (i.e., 100 percent).

Advanced

You can optionally apply basic data filtering to the data displayed on the Gauge. Please refer to the Filtering Columns Table Report section in Creating and Editing Table Reports.

Filter

This basic filtering allows you to further refine which tickets/items to include in the gauge other than the column equal value criteria in the Data Tab. You can optionally apply filtering to the count or sum fields. This filtering allows you to filter which values are used to count/sum fields displayed on the gauge report. For example, you can count the tickets by Priority that are Critical or High.

Create Report

The report is rendered after saving the report. To create the report, click Create.

Note: You can create new reports from existing reports by using the Save As option.

Editing a Gauge Report

ChangeGear allows you to edit a Gauge report and modify the filter criteria and other criteria similar to creating a table report.

To edit a gauge report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Select a Gauge report from the Reports Module grid.

  3. Click the Edit icon from the Gauge tool bar.

  4. You can edit the Description as necessary.

  5. The system allows you to change any report Data Tab or Advanced functions as shown above in the Create a Gauge Report section.

  6. Click Save.

    Note: You can create new reports from existing reports by using the Save As option.
