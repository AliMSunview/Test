Creating and Editing Score Card Reports

In the Report Designer, you can create Score Cards which display the quantity or calculations of tickets/items within a module for a specific time frame, (e.g., current month). The Score Card also shows the percentage of change from the current Score Card counts in comparison with a previous time frame. For example, the Score Card can show the difference between the current month and last month's tickets rendered as a percentage on the Score Card.

The date range (i.e., time frame) is applied to the Score Card using the date range selection list located on the Dashboard or Workspace, or by the date slider located on the dashboard.

To create a score card report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Click the New icon in the reports tool bar.

  3. The Report Creation dialog displays. Select the Score Card report.

  4. Select a module type for the table data used to create the report, (e.g., incident, problem, CMDB, assets, etc.). For example, select the Incident module to create a report regarding Incident tickets.

  5. Select a view for the report, (e.g., All Incidents, Active Incidents, Closed Incidents, etc.). When a Score Card is based upon a view, that report uses the same filter criteria as the view.

  6. Click Next. The Score Card Creation screen displays.

  7. Enter the Name of the report.

  8. Optionally, you can enter a Description for the report.

  9. Select Report Category from the list. The Report Category determines which category within the module to save the report. The My Reports category saves the report in your own My Reports and is not visible to other users.

Advanced

The Score Card does not have a Data Tab. The criteria determining which tickets/items to count is established using the filtering criteria on the Advanced tab. Please refer to the Filtering Columns Table Report section in Creating and Editing Table Reports.

Filter

The filter tab allows you to further refine which tickets/items to include on the Score Card. You can filter which values are used to count tickets displayed on the Score Card report. For example, you can count all the tickets Assigned to your team by setting the filter values to the names of your team members and matching any condition.

Create Report

The report is rendered after saving the report. To create the report, click Create.

Note: You can create new reports from existing reports by using the Save As option.

Editing a Score Card Report

ChangeGear allows you to edit a Score Card report and modify the filter criteria and other criteria similar to creating a table report.

To edit a score card report:

  1. Click Reports on the primary ChangeGear navigation menu.

  2. Select a Score Card report from the Reports Module grid.

  3. Click the Edit icon from the Score Card tool bar.

  4. You can edit the Description as necessary.

  5. The system allows you to change any report Advanced functions as shown above in the Create a Score Card Report section.

  6. Click Save.

    Note: You can create new reports from customized reports by using the Save As option.

Using a Custom Query

You can populate the data set of a Score Card report using a custom SQL query. In order to do this, you must first enable advanced web reporting as outlined in Using the Report Designer. After enabling advanced web reporting, a new Query tab appears in the Score Card report editor.

To use a custom query:

  1. Enable advanced web reporting in the ChangeGear Web configuration file. For more information on advanced reporting, see the Enabling Advanced Web Reporting section in Using the Report Designer.

  2. In the ChangeGear Web client, create a new Score Card report by following the instructions above.

  3. In the Score Card report editor, click the Query tab.

  4. Enter your SQL query in the Query text box. The query should return a single row containing the values that you wish to display. For example, if you wish to display the number of all Incident tickets, your query should return a single row with the total count of Incident tickets.

    Tip: If you would like to see sample queries, open a factory Score Card report in the report editor and navigate to the Query tab.

    • To display trends in Score Card reports, your query should return columns with the following names and attributes:

      • currentCount contains the total number of items for the date range of the report.

      • previousCount contains the total number of items over the previous date range of the report. For example, if the report's date range is for the current month, previousCount displays the count for the previous month.

      • percentChange contains the percentage change from previousCount to currentCount. This should be calculated as a whole number, not a decimal (e.g. "99" instead of "0.99").

    • Optionally, use the $(Filter) variable to insert the criteria in the Filter tab into your query as part of the WHERE clause. For example, the following snippet retrieves tickets that match the filter and are not closed:

      SELECT scorecard.currentCount, scorecard.previousCount, (scorecard.currentCount / scorecard.previousCount) * 100 AS percentChange

      ...

      WHERE {{$(Filter)}} AND [Status] <> 'Closed'

  5. Optionally, check Use drilldown for this report to enable drill down support for the report. Provide your criteria for the drill down in the text box provided. When a user opens the drill down by clicking the report, this criteria will be used to further filter the main query. Drill down will be disabled for this report if the box is unchecked, or if the drill down query text box is empty.

    Important: The drill down criteria should not be a full query, but only the criteria. For example, if you wish to only show new tickets in the drill down, enter "[Status] = 'New'" in the drill down text box.
