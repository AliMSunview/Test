Basic Editor

The Basic Editor lets you build conditions by selecting fields from a drop-down list and defining the values to match for those fields.

  • If you want tickets that match all of the conditions listed, select Match ALL Conditions. For example, setting "Priority = Critical" and "Approved = true" means the ticket must have a critical Priority and be approved before the SLA will apply.

  • If you want tickets that match any of the conditions listed, select Match ANY Condition. For example, setting "Priority = Critical" and "Approved = true" means the ticket can be a critical item or be approved or be both critical and approved before the SLA will apply.

To add a new condition, click the button. To remove a condition, click the button.

Note: You can only choose to match all conditions or match any condition. For more advanced filtering options, see Advanced Editor.
