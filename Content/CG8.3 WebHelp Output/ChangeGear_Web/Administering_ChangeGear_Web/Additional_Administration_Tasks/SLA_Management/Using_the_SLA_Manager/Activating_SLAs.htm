Activating SLAs

SLAs must be activated before they can be applied to tickets. An inactive SLA is not applied to any tickets. If the SLA was previously active before it is deactivated, all tickets that were associated with the SLA will be "orphaned" and no escalations will occur on those tickets. However, the tickets will still be associated with the SLA.

Note: Always keep the default SLA activated. Although the default SLA has no specific selection criteria or resolution time, this ensures that all of your tickets are covered. Any tickets meeting active SLAs other than the default SLA take precedence over the default SLA.

To activate an SLA:

  1. Open the SLA Manager.

  2. Select the SLA you wish to activate.

  3. Select the Activate SLA check box next to the Name field.

  4. Save the SLA.

To deactivate an SLA:

  1. Open the SLA Manager.

  2. Select the SLA you wish to deactivate.

  3. Clear the Activate SLA check box next to the Name field.

  4. Save the SLA.
