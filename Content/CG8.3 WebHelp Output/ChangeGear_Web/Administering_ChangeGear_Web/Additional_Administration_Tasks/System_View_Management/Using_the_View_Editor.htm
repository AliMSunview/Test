Using the View Editor

The Manage View menu normally allows you to create a custom view from existing views. Administrators can edit system and custom views by invoking the View Editor, (i.e., select the Edit option from the Manage View menu in the tool bar).

When editing a view, you may select whether the view is visible in the ChangeGear module navigation menu. The View List, located in the grid tool bar, allows you to identify all system views and custom views available for a ChangeGear module.

Note: You can only delete a system view using the ChangeGear Desktop client application. You may delete custom views using the Manage View menu option on the web.

To enable the view editor for system view maintenance via the Web:

You can enable the View Editor for system view maintenance on the web by utilizing the Administration module of ChangeGear Desktop Client application. To enable the View Editor, select the Administration object of the Role Details dialog and enable the Edit privilege. Refer to the ChangeGear Privileges section of the Administrators Guide for an explanation of role and privilege options.

To create system views via the Web:

You may create system views on the web using the Save As button available using the View Editor, (i.e., the Edit View option located on the Manage View menu). When Using the View Editor, any new view created from an existing system view remains as a system view upon save. Any new view created from a custom view remains a custom view upon save.

Note: Selecting the Save As New View option on the Manage View menu in the grid tool bar always creates a custom view type.

  1. Select a system view to display in the grid either by using the module navigation menu or using the View List in the grid tool bar.

  2. You can create a system view by selecting the Edit View option to make modifications to an existing system view.

  3. Use the Save As button in the View Editor to create a new view.

To edit system views via the Web:

You may edit system views by performing the same steps as explained in Editing Custom Views using View Editor on the web.

To move a view in the Navigation Group menu:

You can move a system view to any view group, (e.g., My Views, My Team Views, Global Views), by changing the Ownership tab.

In addition, the Ownership tab determines the scope of the data displayed in the grid. For example, when a team is specified as the Owner, Requestor, or Assigned to team for an incident, and the view displaying the grid data specifies User's Team, the grid shows only incidents associated with the User's Team.

  1. Select the system view from the View List.

  2. Select Edit View from the Manage View list.

  3. Select the Filter tab.

  4. Select the Ownership tab.

  5. Select the group in which you want the view to appear in the navigation menu.

    • All Viewable - Appears under Global Views.

    • User's Team - Appears under My Team Views.

    • User's Own - Appears under My Views.
