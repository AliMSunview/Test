Designing a New SSP

The Self-Service Portal (SSP) is the primary interface for end users to create tickets, view submitted tickets, and access knowledge base articles. Customizing the SSP allows you to control the appearance and style of the page, as well as the functionality available to end users.

To design a new SSP, or to redesign an existing SSP:

  1. From the Self-Service Portal slide-out, click the view image button  in the upper right corner. ChangeGear opens the Customize  designer panel in a new tab.

  1. ChangeGear populates the Status field based on when you publish the SSP you are working on. Until you publish, the SSP status is Draft.
  2. The SSP designer provides code-less customization controls to configure the layout and personalize widgets. The customization panel of the SSP provides a responsive page layout allowing the choice of fit to screen, scroll vertical or horizontal, or simply a fixed position.

The following are the layout types:

  • Fit to Screen – This is the default and recommended layout type which auto resizes the widgets to fit them in the screen without scroll.
  • Scroll Vertical:  Auto resizes widgets to fit within the horizontal screen size but does not auto resize as widgets are moved vertically as the page will now allow a vertical scroll. 
  • Scroll Horizontal: Auto resizes widgets to fit within the vertical screen size but does not auto resize as widgets are moved horizontally as the page will now allow a horizontal scroll.

Note: Scroll Vertical or Scroll Horizontal layout types are recommended for SSPs where all the widgets do not fit well in the screen and scrolling is needed. The option to use horizontal or vertical is based on preference and how you want your widgets to look and flow.

  • Fixed: Does not auto resize widgets and will allow both vertical and horizontal scrolling.  This is recommended in scenarios where you may have a lot of content, but also many ways of viewing said content.  Different devices with different screen sizes and resolutions could cause issues, fixed widget sizes could help improve the experience.

Warning: Changing between these layout types can require adjustments, re-positioning, and/or redesigning of the widgets.

  1. In the Widgets section, choose the type of widget you want to add: HTML, Links, Tickets, Views, Search, or Button.

HTML Widgets

  1. For HTML widgets, click the button. ChangeGear adds the HTML widget on the workspace. Resize the widget and position it where you want it to appear in the SSP. You can drag it to move it around and arrange it as desired. To make it larger or smaller, drag the corner.  
  2. Click to configure the widget. ChangeGear displays the Settings - HTML screen.

  1. Enter the Title and click the dropdown on the right to choose a color for the title bar.
  2. Click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to open the Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  3. Next, check Display as banner if you want to use this HTML widget without the HTML header.
  4. Next, design your content using WYSIWYG editor or choose the custom button and add your html content in the content area . This area is very customize-able, for example you can also use JavaScript or CSS.

Note: Switching between options will result in loss of content. User will be presented with warning to confirm with switching and clearing content. The WYSIWYG editor is available from ChangeGear release 8.3. Existing HTML widgets will default to ‘Custom’ editing option so as not to lose content while New HTML widgets will default to using the wysiwyg editor.

  1. Finally, click Apply to save your settings to the HTML widget.

Links

The Links widget redirects users to other areas of ChangeGear or to other websites.

  1. For Links widgets, click the  button. ChangeGear then adds the Links widget on the workspace. 
  2. Click in the center of the widget to configure it. ChangeGear displays the Settings - Links screen.

  1. Enter the Links widget title and select the background color.
  2. Click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to open the Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  3. Check Show divider lines if you want to display lines in-between each link.
  4. Check Show bullet points if you want to display the links as bullets.
  5. To add links, click the plus sign to the right of Links. ChangeGear opens a row in the links grid.
  6. Enter the Link Text and Link URL. Continue adding links as desired.
  7. Click Apply. When users click on a link in the list, ChangeGear opens the page in a new window outside of ChangeGear.

Add Tickets Widget

The Add Tickets widget allow users to view tickets.

  1. For Add Tickets widgets, click the  button. ChangeGear then adds the Add Tickets widget on the workspace. 
  2. Click in the center of the widget to configure it. ChangeGear displays the Settings - Add Tickets screen.

  1. Enter the Links widget title and select the background color.
  2. Click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to open the Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  3. Check Show divider lines if you want to display lines in-between each ticket.
  4. Check Show bullet points if you want to display the tickets as bullets.
  5. In Modules, select the type of module ticket you want to display. This dropdown is tied to the modules for which you are licensed.
  6. In Views, select the type you want to display:  All Active, All Closed, All New, All, or Assigned to Me.  Note: this list will vary depending on the module type you selected.
  7. In Number of rows to display, enter how many rows you want presented to the user.
  8. In Display Field 1, select the first field you want to display--for example ItemID.
  9. In Display Field 2, select the second field you want to display--for example Summary.
  10. In Form Name enter the form name.
  11. Click Apply.

Search Widget

The search widget allows users to search multiple modules simultaneously. It also offers type-ahead functionality, which provides suggestions to users as they enter their queries. You can specify the entities that appear in the results, as well as the text that is displayed if the search does or does not return results.

  1. For Add Search widgets, click the  button. ChangeGear then adds the Add Search widget on the workspace. 
  2. Resize the widget and position it where you want it to appear in the SSP. You can drag it to move it around and arrange it as desired.

Add Button Widget

The button group widget displays a grid of individual button controls that users click on to perform actions such as submitting tickets and retrieving contact information.

  • Ticket Buttons -- allow users to create and submit new tickets.
  • Link Buttons -- redirect users to other areas of ChangeGear or to other websites.
  • Service Request Button -- allow users to create and submit new service requests.
  1. For Button widgets, click the  button. ChangeGear then adds the Add Button widget on the workspace. 
  2. Click in the center of the widget to configure it. ChangeGear displays the Settings - Button screen.

  1. Enter the Title and Description for the button.
  2. Click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to open the Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  3. Click Hide Text to display an image only, without text.
  4. Click Hide Image to not display the image.
  5. Click Remove Image Scaling so ChangeGear ignores the aspect ratio on the button.
  6. Click Background color to set this for the button.
  7. Click Types to select if this button is a Link, a New Ticket, a Service Request, or HTML. The fields below change depending on what button type you select, to allow you to enter further settings.
  8. Click Apply.

Willow Widget

Willow is a chat-style response system that helps users troubleshoot and resolve incidents without having to involve IT staff. End users and staff users access Willow by opening the SSP and clicking on Willow's icon. Note: For more information on Willow, please see the Using Willow topic in the Self Service Portal help.

  1. For Willow widgets, click the  button. ChangeGear then adds the Willow widget on the workspace. 
  2. Click in the center of the widget to configure it. ChangeGear displays the Settings - Add Widget screen.

  1. Select the Model from the dropdown and add a Description.
  2. Click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to open the Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  3. Choose a Background color for Willow.
  4. Click Apply.

Trending Widget

The configuration set here determines what is displayed in the Trending widget. The default Trending configuration includes basic factory settings, but you have the option to change the settings as described below:

On the newly added Trending widget, click on “Click to Configure” to display the Settings – Trending window, and make (adjust/edit) the following settings:

2.  Enter a title that appears on the Trending widget.

  • Select a color for the Title lettering.
  • Select an Icon from the Icon Library or upload/drag and drop an icon image file.
  • Enable the bullet point option if you want trends to appear in the form of bullet points. Enter the number of trends you want to display on the widget.
  • Enter the number of days from which you want to see trends.

Note: Up to 10 trends can be displayed on the widget, up to the last 365 days.

3.  Click Apply to save the settings.

4.  In the Customize slideout, click to store the settings, and /or click to publish the new Trending widget.

Luma Widget

Luma Virtual Assistant (Luma VA) provides a conversational interface driven by Natural Language Processing (NLP) that improves self-service drastically and guides conversation to trouble shoot and resolve issues quicker without having to involve IT staff. It also allows for capturing details and open actionable requests directly from your chat.

Luma Virtual Agent (VA) is the industry’s first knowledge-centered virtual agent for streamlining enterprise service and support. Luma VA leverages knowledge functionality to harness the power of information and make it consumable.

To add Luma VA to your SSP:

  1. Click the button which adds the Luma widget on the workspace.
  2. Click the center of the widget to configure it. ChangeGear displays the Settings – Luma screen.

  1. Enter the name (can be a friendly name or “Luma”).
  2. Enter the URL and the secret key to access Luma VA. Please check with your ChangeGear administrator to get this information.
  3. The icon will default to Luma icon . To customize, click Upload to add an image to the widget.
    You can also drag and drop an image into place. 
    – OR-
    Click Icons to     Icon Library and select an icon. Scroll to review and select the icon you want added to the widget. 
  4. Background color and Luma accent color are set to their default color. To customize, click and choose the desired color.
  5. Click Apply to save the settings.

Popular Solutions

The Popular Solutions control displays a list of the four highest rated articles from the Knowledge Base. Clicking on an article opens the article in a new window. You can also click on the title of the control to access the KB module, where you can also view the most recently published KB articles as well as search all published KB articles.
