Multiple Self-Service Portals

The ChangeGear Self-Service Portal (SSP) provides a quick, easy-to-use tool for reporting, updating, and tracking incidents, finding help in Knowledge Base articles, finding quick links to other resources, and more. The Self-Service Portal can be highly customized. Depending on the ChangeGear configuration, end users can easily submit incidents to their IT department at any time without sitting on hold for a technician or navigating through complex phone menus. 

ChangeGear allows you as an administrator to set up multiple SSPs for your end users. You may for example want to set up an SSP for each company location, or for each department within the company--and adjust the SSP accordingly for each end user group.

SSPs can be global (all users can see them), or be set to one of these types: location, department, role, or team. Only users that belong to a specific location, department, role or team can see those SSPs. For example, if you set up a separate SSP (of type department) for your Human Resources department, only the employees within that department can view the HR SSP. Or, if you set up SSPs for each office location, only end users whose location in ChangeGear matches that location can view that SSP.

ChangeGear supplies a global SSP that includes everyone in the company as a default. If a department or location does not have its own individual SSP, ChangeGear instead displays the default to those users.

Each SSP has a status of either Draft or Published; ChangeGear does not show an SSP to users until its status is Published. You can edit and refine an SSP and its design until you are ready to 'go live' by publishing. Also, if you change an existing and published SSP, ChangeGear does not show the updated SSP until you re-publish it.
