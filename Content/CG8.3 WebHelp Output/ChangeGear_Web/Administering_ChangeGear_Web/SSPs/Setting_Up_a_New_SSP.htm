Setting Up a New SSP

To add a new SSP:

  1. Click Web Administration>SSP. ChangeGear displays the Self-Service Portals screen.

  1. ChangeGear lists the Default global SSP.
  2. To add a new SSP, click the plus sign  in the upper right corner. ChangeGear opens a slide-out editor.

  1. Enter the Name and Description for the SSP you are adding.
  2. In the Type field, set the type of the new SSP as to Global (all users can see them), or one of these types: Location, Department, Role, or Team. Only users that belong to a specific location, department, role or team can see those SSPs.
  3. If you want to set your new SSP as the default, first publish it and then check the Set as Default box. All users who are not set up in the Department, Location, or Team groups will see the default SSP.
  4. The Link field shows the URL for your new SSP. You can copy this link for sharing with others, although if the SSP is a group and the user is not in that group, they will not see a 'not authorized' message if they click on the link.
  5. You have now set up a new SSP. To design what widgets are on the SSP layout and how they look, refer to Designing a New SSP.

Copying an Existing SSP

As a shortcut to creating a new SSP from scratch, ChangeGear also lets you copy an existing SSP and then edit the copy. This saves steps, especially if the copied SSP is similar to an existing SSP.

To copy an existing SSP:

  1. Hover over the SSP you want to copy, and click the More Actions ellipsis  on the right. Then select Copy Row.

  1. ChangeGear copies the SSP and pastes it into the grid. ChangeGear assigns the new SSP a Draft status.

  1. You can now edit the new SSP by clicking on the pencil icon for that row.

Note:  You can review an existing SSP's layout to help in your decision to copy it, by clicking the More Actions ellipsis  and then selecting View Layout. ChangeGear opens the SSP in another tab.
