Editing Custom Views Using the View Editor

You can edit your custom views using the View Editor.

To edit custom views:

  1. Select a ChangeGear module, (e.g., incident, problem, CMDB, assets, etc.), from the primary ChangeGear Navigation Menu. For example, select the Incident module to edit or create a custom view used for displaying tickets on the incident grid.

  2. Select the view you want to edit OR create a new view from an existing view using the View List in the grid tool bar.

  3. Open the Manage View menu on the grid tool bar.

  4. Select Edit View, the View Editor dialog displays.

    Note: The Edit View option is grayed out for system views unless the user has Administrator privileges.

  5. Optionally, you can edit the Description for the view.

Column Selection

The Column Selection tab already highlighted in the View Editor dialog is where you can specify which columns to display in the view. The dialog shows the Fields Selected for the current view. 

  1. To add fields to Fields Selected for the view, select from the Fields Available and click the arrow pointing right . To select all available fields, click the double arrows pointing right . Similarly, you can remove fields from the Selected list, by clicking the arrow(s) pointing left .

  2. To arrange the order of the Fields Selected on the view, select the field and press the up arrow or down arrow .

  3. Select a field from the Fields Selected list. The following parameters apply to the field you selected:

  • Use the Width parameter to size the column length in pixels on the grid.

  • The Caption parameter allows you to use a different name for the column heading displayed on the grid.

  • The Display Format field controls the appearance and format of date and/or number fields on the report. The Display Format field is visible only when you have selected a date or a numeric field such as cost. In addition to the formats shown below, you can use the date and time formats specified in the Windows Control Panel > Region and Language setting such as M/d/yyyy, MM/dd/yy, MM/dd/yyyy. Type in the format as you see it, (e.g., MM/dd/yyyy,). The format is case sensitive.

Date Formats

d

Represents the short date format in the US standard format; MM/DD/YYYY.

For example, if the field contains value 6/15/2009 1:45:30 PM, the formatted date on the report displays as 6/15/2009.

g

Represents the general short date and time in the US standard format; MM/DD/YYYY HH:MM.

For example, if the field contains value 6/15/2009 1:45:30 PM, the formatted date on the report displays as 6/15/2009 1:45 PM

Numeric Formats

c

Represents currency format and specifies decimal places in the US standard format. Positive numbers are formatted as $.00. Negative numbers are formatted as ($.00).

For example, if the field contains value 123.456, the formatted dollar amount on the report displays as $123.46. If the field contains value -123.45, the formatted dollar amount on the report displays as ($123.45).

d

Represents integer digits and can be positive or negative.

For example, if the field contains value 1234, the formatted value on the report displays as 1234. If the field contains value -1234, the formatted value on the report displays as a negative -1234.

n

Represents integer and decimal digits using "," group separators, and a decimal separator with optional negative sign.

For example, if the field contains value 1234.567, the formatted value on the report displays as 1,234.57. If the field contains value -1234.56, the formatted value on the report displays as -1,234.56.

NOTE: n2 works the same as n.

Filtering Columns

You can optionally filter the data you want displayed from the view.

  1. Select the Filter tab in the View Editor dialog.

Basic Tab

The Basic Tab allows you to do simple filtering by selecting fields and required values.

  1. Click the Basic tab to apply basic data filtering. Select data match conditions using the radio buttons.

    • Match All Conditions means the data must meet each one (all) of the filtering options specified to appear in the view result set, (i.e., acts like an AND condition for filtering options).

    • Match Any Condition means the data can match any one of the conditions and is displayed in the view result set, (i.e., acts like an OR condition for filtering options).

  2. Press Select field. Select All to view all the fields available and select the proper field for match condition. For example, scroll down the fields to select the Status field.

  3. Key in a conditional value for filtering data. When the field has fixed valid values, a list shows the available values.

    Tip: You can add a date range for filtering any date fields you have selected. You may enter $(DATE_RANGE) in the value field for the date you are filtering. Next, you can navigate to the Date Range tab to select a date range.

  4. Press Add to complete filter criteria. Press to delete criteria. You can add multiple filtering criteria as necessary for your view.

  5. If finished entering filter criteria, click the Create, or continue filter criteria by clicking another tab choice, (e.g., Advanced, Ownership, Date Range.)

Advanced Tab

The Advanced tab allows you to create more complex filtering criteria using SQL-like queries.

  1. Click the Advanced tab to code more complex queries. Any filtering criteria you have entered in addition to the selected view filtering criteria is shown here. Add or change query as necessary for your view.

    Note: The system warns you when switching to the Basic filtering tab the Advanced filtering criteria is removed.

Ownership Tab

The Ownership tab allows you to limit the scope of the data displayed for the view based on what the user owns as well as what the user's team owns. Custom views appear under My Views group menu.

  1. Click the Ownership Tab and select from the following:

    • All Viewable: All data for the view is visible to users on the grid, (ownership and team membership have no impact).

    • User's Team: Only data for the user's team displays on the grid for this view when the team is the Owner, or the Assigned to, or the Requester.

    • User's Own: Only data for which the user is the Owner, Requester or the Assigned to user, is visible on the grid for this view.

Date Range

The Date Range tab enables you to create a default date range for your view date filter criteria and is used in conjunction with a value used in filtering criteria. Any dates used for filtering in the Basic tab or the Advanced tab can use the default date range.

  1. Click the Date Range tab to set a default date range.

  2. Select a Date Range from the list of date ranges, (e.g., current month, last 6 months, etc.). The $(DATE_RANGE) variable is set to the default date range selected.

  3. Click the Advanced tab. The code in this tab shows the variable $(DATE_RANGE) used as filtering criteria for the CreatedDateTime field filtering. For example, if you selected current month as your default date range on the Date Range tab, the data displayed in the grid shows only those tickets created within the current month.

Grouping Columns

The Grouping tab in the View Editor dialog allows you to optionally group the data rows displayed for the view by one or more columns.

  1. Select from the Fields Available for the desired grouping on the grid. You can select multiple fields at once by holding down the CTRL key.

  2. To add selections to Fields Selected for view grouping, click the arrow pointing right . To select all available columns, click the double arrows pointing right . Similarly, you can remove fields from the Fields Selected list, by clicking the arrow(s) pointing left.
  3. To arrange the order of the Fields Selected on the grid, select the field and press the up arrow or down arrow .

Sorting Columns

Sorting items in the view organizes your grid data rows in a meaningful way. For example, sort by the Created date column to view tickets in oldest to most recent ticket order.

  1. Select from the Fields Available the fields you want to use for sorting the view data. You can select multiple columns at once by holding down the CTRL key.

  2.  To add fields to the Fields Selected for view sorting, click the arrow pointing right . To select all available fields, click the double arrows pointing right . Similarly, you can remove fields from the Fields Selected list, by clicking the arrow(s) pointing left.

  3. To arrange the order of the Fields Selected in the view, select the field and press the up arrow or down arrow .

  4. Select sort sequence by clicking on a Selected item and check the box ascending sequence or no check for descending sequence.

General Tab

The General tab allows you to limit the views visibility in the Navigation Menu and the View List based on what the user owns as well as what the user's team owns.

  1. Click the General Tab and select from the following criteria. You can select multiple criteria as required.

    • Visible: The view is visible to all users on the View list, (ownership and team membership have no impact).

    • Appears in Navigation Panel: The view is visible in the Navigation Menu under the appropriate category.

    • Hide from Users with View Own privilege only: Only views owned specifically by the user are visible on the View list.

Save View

To render the view data on the grid, save the view by clicking Save.

Note: You can create new views from an existing view using the Save As option.
