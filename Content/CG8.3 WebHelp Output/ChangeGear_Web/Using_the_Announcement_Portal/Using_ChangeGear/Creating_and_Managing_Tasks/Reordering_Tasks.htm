Reordering and Grouping Tasks

After you add tasks to a ticket, you may need to change the order in which certain tasks are listed. ChangeGear allows you to define the order in which tasks must be implemented. Depending on your ChangeGear configuration, this also ensures that a task cannot be started before its preceding tasks have already been completed. Tasks that have a lower priority number appear first on the list, while tasks with a higher priority number appear further down the list. Tasks that do not contain an order appear at the end of the task list and can be worked on at any time.

When a task is either promoted (moved to a lower order number) or demoted (moved to a higher order number), it is automatically grouped with the tasks sharing the same order number. For example, moving a task from order 3 to order 2 groups the task with other tasks in order 2. If this results in an empty group, then ChangeGear automatically reduces the order of all subsequent tasks by one (1) to account for the missing group. Tasks within a group can be completed in any order.

To reorder a task:

  1. Open the ticket.

  2. In the Implementation or Tasks tab, select the task that you want to reorder.

  3. In the tab toolbar, click the Move Up or Move Down button to change the order of the task. The task automatically groups with the order that it's moving in to. If the task was the last one in its group, then all subsequent tasks will adjust their order number down by 1 to account for the lost group.

    • Alternatively, click the Move to Top or Move to Bottom buttons to move the task directly to the beginning or end of the list.

  4. Click Save.

Note: If you wish to enforce that tasks are worked on in a certain order, see the topic Managing Task Dependencies.
