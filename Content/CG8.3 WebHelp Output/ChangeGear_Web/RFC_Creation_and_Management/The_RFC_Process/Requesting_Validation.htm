Requesting Validation

Some change management systems require change owners to submit RFCs for validation before closure. The original requester reviews the implementation against the RFC. If the RFC meets the criteria for success, the requester validates and submits the RFC to the Pending Close status.

The best way to ensure that your implementation passes the criteria for success is to document the completion of every requirement. When you are satisfied that your implementation is complete, you can submit the RFC for validation.

To request validation:

  1. Review the Implementation tab to ensure that it accurately reflects the following information:

    • Actual start and completion dates

    • Actual start and completion times

    • Duration in hours

    • Total effort and labor costs

    • Total resource costs

  2. Click the Release tab and ensure that both All tests were successful check boxes are selected.

  3. Once all tasks in the Implementation tab are complete and saved with a Closed status, click the Request Validation button.
