Reviewing the RFC

Once the change request is implemented and deployed, the change needs to be validated. Validation is the measurement of a change request against its stated benefits, its actual costs, and its impact. This phase of the change management life cycle determines whether the change request succeeded or failed, as determined by the requester.

If a change request meets the criteria for success, the requester can request to close the request using the Requester Validated option from the toolbar. However, if the request does not meet expectations, the requester selects the Failed Validation option and the request goes back to the Assigned flow step.
