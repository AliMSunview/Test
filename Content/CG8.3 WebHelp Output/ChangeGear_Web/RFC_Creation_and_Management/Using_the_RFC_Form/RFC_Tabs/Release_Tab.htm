Release Tab

To document the testing and deployment phases, ChangeGear provides a Release tab that you can use to manage and document change process testing and deployment. For both the Quality Assurance and the Deployment sections, you can attach test case and test result documents in the Attachment tab, as well as indicate tests were successful and when the deployment is successful.

Tip: Ideally, you'll simulate the change in the test environment. After the simulation succeeds, you can implement the change in the production environment.

To assign QA and Deployment Owners:

  1. Open an RFC.

  2. Click the Release tab.

  3. Determine the type of test-related processes you want to assign, either QA or Deployment, or both.

  4. In either of the sections, click the Owner drop-down and select an Owner.

  5. Type in any comments in the Comments field.

  6. If you were assigned to verify QA or Deployment processes, you can click the All tests were successful (for QA) or Successfully Deployed (for Deployment) to indicate successful completion of these processes.
