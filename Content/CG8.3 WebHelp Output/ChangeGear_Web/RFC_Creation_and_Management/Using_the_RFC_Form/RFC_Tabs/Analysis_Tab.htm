Analysis Tab

This tab supports the review phase of the change management process. You can use this tab to reassess, expand, and refine the requester's analysis.

Note: ChangeGear allows you to attach documentation related to analysis requirements. You can add three types of attachments in the Analysis tab; testing, rollback, and training. In the Release tab you can add test case and test results attachments. If you add attachments through the Analysis tab or the Release tab, the field column Attachments list displays. To add an attachment, click the Attachments tab and select the file using the Add Attachment dialog box, or click the paperclip icon next to the requirements.

To specify Analysis details:

  1. On the RFC form, click Analysis tab.

  2. Clear the Testing and the Rollback Plan check boxes if the RFC does not require testing or a rollback plan.

  3. Select the Impl. Plan, User Training, Planning, or Downtime check boxes if any of these items are required.

    Tip: If you selected any of the items in Steps 2 and 3, you can attach supporting documentation by clicking paperclip icon.

  4. In the Plan section, enter the planned start and complete dates and times.

  5. In the Planned Duration box, type the estimated total number of hours the change requires.

  6. In the Planned Effort box, type the estimated total number of person-hours the change requires

  7. In the Alternative Solution(s) box, type a detailed description of any alternative solutions. (If you have supporting documentation for the alternative solution, click the Attachments tab to add it.)

  8. In the Risk field, select the level of risk carried by the RFC.

  9. In the Possible Side Effects field, type a description of any side effects that the RFC may produce.

Note: For more information on adding attachments, see Managing Attachments.
