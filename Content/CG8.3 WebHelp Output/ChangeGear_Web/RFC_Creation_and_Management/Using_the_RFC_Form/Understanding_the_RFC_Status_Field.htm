Understanding the RFC Status Field

The status field of the RFC indicates where the ticket is in its life cycle. The default setting for a new ticket is New and the status when completed is Closed. When manually changing the status, the list you may choose from depends upon the current status of the RFC and your ChangeGear role. A requester has a limited set of permissions.

Status

Description

Assigned

Either the change manager or an assigned reviewer has assigned the RFC to a change owner for implementation.

Closed

RFC requires no further consideration and is closed.

Escalated

RFC is put in a temporary hold state until appropriate approval can be obtained.

In-Progress

Assignee of the RFC has begun the implementation of the request.

Need More Info

RFC requires more information before it can be implemented.

New

RFC is created and submitted for approval.

On-Hold

The implementation of the RFC cannot proceed.

Pending Approval

RFC is awaiting approval from all its approvers.

Unsubmitted

RFC is created but not yet submitted.
