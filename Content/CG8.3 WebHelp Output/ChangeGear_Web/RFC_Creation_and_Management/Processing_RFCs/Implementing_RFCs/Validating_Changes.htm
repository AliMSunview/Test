Validating Changes

Validation is the measurement of a change request against its stated benefits, its actual costs, and its impact. No change request is complete without validating the change tasks.

Once all tasks in the Implementation tab are complete and saved with a Closed status, the change task owner requests validation. When validation is requested, the original requester receives an e-mail notifying them the change request is complete and to please verify. The requester verifies the change. If there are issues discovered as a result of the change, a rollback plan is initiated. This rollback plan lets you back out of a change request if it results in failure.
