Rollbacks

Even if a change request includes very detailed plans, unexpected results can occur. Every major or significant change request should include a rollback plan. This allows you back out of a change request if it results in failure. You should be able to return the resource to its previous configuration and respond to the failure with immediate communication and documentation.

ChangeGear’s RFC form lets you attach documented rollback plans for the change request. You can attach the rollback plan to the RFC, allowing you review it while you analyze the change request. During implementation, you can find the rollback plan in the RFC form by displaying the Analysis tab or the Attachments tab.
