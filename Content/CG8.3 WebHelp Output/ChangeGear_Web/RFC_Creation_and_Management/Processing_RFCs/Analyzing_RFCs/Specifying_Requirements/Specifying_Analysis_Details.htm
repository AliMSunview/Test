Specifying Analysis Details

The Analysis tab, located on the lower portion of the RFC form, provides several options for your RFC. You can select or clear requirements as needed.

Note: ChangeGear allows you attach documentation related to analysis requirements. To do this, click the appropriate button and select the file using the Add Attachment dialog box.

To specify analysis details:

  1. On the RFC form, click Analysis.

  2. Check the Testing, Rollback Plan, User Training, Planning, Implementation Plan, Downtime check boxes when any of these items are required.

  3. In the Start boxes, enter the planned start date and time.

  4. In the Complete boxes, enter the planned completion date and time.

  5. In the Duration box, type the estimated total number of hours the change requires.

  6. In the Effort box, type the estimated total number of person hours the change requires.

  7. In the If Not Implemented box, type a detailed description of the potential consequences of not completing the RFC.

  8. In the Alternative Solution(s) box, type a detailed description of any alternative solutions. (If you have supporting documentation for the alternative solution, click the Attachments tab to add it.)

  9. In the Possible Side Effects box, type a description of any side effects that the RFC may produce.

Note: For more information on adding attachments, see Managing Attachments.
