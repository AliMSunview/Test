Overview of Specifying Requirements

Typically, RFCs require testing and rollback plans prior to implementation. The Require group, in the upper-left corner of the Analysis tab, provides several requirement options for the RFC. You can select or deselect the RFC requirements as necessary.

Tip: ChangeGear automatically selects the Testing and Rollback Plan check boxes. You would only select these check boxes if the requester or another user previously cleared them. Similarly, if the RFC does not require testing or a rollback plan, you can clear these check boxes.

To set RFC requirements:

  1. Open the RFC for which you want to specify requirements.

  2. Click the Analysis tab.

  3. Select the Testing check box if the RFC requires testing.

  4. Select the Rollback Plan check box if the RFC requires a rollback plan.

  5. Select the User Training check box if the RFC requires your organization’s members to acquire new skills.

  6. Select the Planning check box to indicate whether the change request is planned or unplanned.

  7. Select the Implementation Plan check box if the RFC requires an Implementation plan.

  8. Select the Downtime check box if the RFC will requires down time.

  9. Save your changes to the RFC form.
