Attaching Requirement Documentation

The Requirements group also lets you attach files supporting your testing, rollback plan, implementation plan, planning and user training requirements. The documents you attach to the Analysis tab can provide instructions for completing the requirements. When you attach files to your change requests, it is unnecessary to look elsewhere for support documentation. Afterwards, anyone who requires detail can access the pertinent files through the RFC form.

To attach requirement documentation:

  1. Click the Analysis tab.

  2. In the Require group, click the appropriate attachment button.

  3. In the Add Attachments dialog box, navigate to the appropriate directory location of the file you want to attach.

  4. Select the file that you want to add and click Open.

  5. If you want to link your attachment instead of copying it to your RFC, click Save As Link.

    Note: When you provide a link URL as an attachment to the RFC, any changes or updates to the source file are automatically updated in your RFC. Or, if you attach the actual document, any changes to the source file have no effect on your attachment. This is an effective way to create an archive of your test plans, etc.

  6. In the Type list, select the type of file and click OK.

  7. Save your changes to the RFC form.
