Deploying Changes

The deployment of an RFC is where existing resources or services are altered in some way. Because these changes will create a reaction, you should announce the deployment or release of the RFC to everyone affected by the impact of these changes.

You can assign a change owner to the Deployment process and attach an implementation plan from the Deployment section of the Release tab. The owner uses this tab to confirm the implementation plan was successful using the Successfully Deployed check box.

To assign a deployment change owner:

  1. Create a new, or locate an existing RFC, and click Release.

  2. In the Release tab under Deployment Owner, click the icon.

  3. In the Select User dialog, select the user, team, role, dynamic role, or contact you wish to use as the deployment owner.

  4. Click OK.
