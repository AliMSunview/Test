Overview of Testing RFCs

Many RFCs require testing as a safeguard against undesirable outcomes. To determine if an RFC requires testing, look in the Require group of the Analysis tab.

If you are the change owner or change manager, you can add testing tasks to the Implementation tab as necessary. If you are implementing an RFC, check the Implementation tab to see if any testing tasks are included.

Document tests using test cases and results to perform before implementation in the Quality Assurance (QA) group of the Release tab. You can also test an RFC after implementation to gauge its success. If your RFC is particularly complex, you may need to conduct pre- and post-deployment tests.
