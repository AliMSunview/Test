Performing Quality Assurance

Part of the Quality Assurance (QA) process is testing a change request before it is implemented. You can assign a change owner to the Quality Assurance process and attach test cases and test results from the Quality Assurance section of the Release tab. The QA owner uses the Release tab to verify the test cases and test results. The owner uses the Release tab to confirm these tests were successful using the All tests were successful check box.

To assign a QA change owner:

  1. Create a new, or locate an existing RFC, and click on the Release tab.

  2. In the Release tab under Quality Assurance, click the icon.

  3. In the Select Items dialog, select the user, team, role, dynamic role, or contact you would like to enter as the QA Owner.

  4. Click OK.
