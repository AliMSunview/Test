The Testing State

In this stage, two types of tests are conducted. First, user acceptance tests are performed to verify that the changes implemented are in alignment with RFC requirements. Second, installation procedures, including pilot testing in the production environment and rollback procedures are tested. To ensure segregation of duties, testers should be independent from the staff developing the change. At this stage, the Deployment owner finalizes the implementation plan in preparation for the Deployment stage. Change implementers continue to update and close tasks as appropriate.

To advance to the Testing State:

  1. Open the RFC.

  2. In the toolbar, click Verify.

To return to the Assign State:

  1. Open the RFC.

  2. In the toolbar, click Cancel.

Test Cases

Storing test cases and test results in ChangeGear streamlines the testing process. During the planning stage, the release manager assigned the quality assurance owner to oversee the quality assurance process, including development of test cases. Early in this stage or in the Planning state, you should upload the test cases.

Test Results

The results of testing are essential to the release management process. After conducting tests, Quality Assurance uploads the Test Results. In addition, the QA owner notifies the Release Manager and relevant stakeholders about the results of the tests.

Uploading the Implementation Plan

In preparation for the deployment stage, the Deployment owner should upload the Implementation Plan as a reference for staff involved in performing and overseeing the implementation activities.

Communicating Deployment Plans to the User Community

According to test results, appropriate staff and stakeholders are notified of upcoming deployment activities.

If all functionality tests and pilot tests of deployment into the production environment were successful, the Release Manager and the Change Manager validate the release for deployment.

When Testing Fails

If the tests failed to provide the desired results, the Release Manager in conjunction with the Change Manager and relevant stakeholders, may choose to:

  • Click the Failed Testing button to return to the build state to fix the outstanding problems.

  • Click the Defer Release button when there are problems preventing successful deployment of changes.

  • Click the Cancel button for release until further testing can be performed.
