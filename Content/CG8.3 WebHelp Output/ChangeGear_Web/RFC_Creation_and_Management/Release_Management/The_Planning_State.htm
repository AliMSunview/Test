The Planning State

Using the information gathered in the Release stage, the release manager starts planning the release in greater depth and detail. For instance, any missing information is obtained and implementation procedures are divided into tasks with each task assigned to the appropriate team or person.

The implementation time line is defined and a rollback plan is drafted. In this stage, it is recommended to consult the configuration management database for information on configuration items (CIs) affected by the change to inform planning and implementation strategies. Another best practice is to identify reviewers for the testing stage, allow testers enough advance notice to develop suitable test cases.

To enter the Planning State:

  1. Open a RFC in the Release state.

  2. Review the RFC to understand the general aspects of the change request. For example, determine the scope and requirements, assess the impact and risk, or review the prospective time line.

    • When you have reviewed the RFC and feel comfortable with continuing the release process, select the Setup option.

    • When you have reviewed the RFC and do not feel comfortable with continuing the release process, click Cancel Release. Add any relevant information to the ticket for further processing.

  3. At this time, fill in the QA and Deployment information in the Release Tab, including supporting information such as test cases or implementation plans. Create and assign implementation tasks.

Quality Assurance and Deployment Owners

A structured approach to change and release management requires segregation of duties and ownership of processes. Consequently, it is important to ensure that there are people in charge of critical aspects of the RFC life cycle, such as quality assurance and deployment tasks. Ownership does not imply that one person should perform every task in the change request.

Supporting Documentation

Adding supporting documentation to the RFC provides a central repository of resources to guide, assess, and supplement the release process.

Creating and Managing Tasks

Tasks enable you to create a prioritized time line of activities in alignment with change request requirements. Each task is assigned to an implementer who provides a detailed account of implementation activities.
