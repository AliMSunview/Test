The Deployment Approval State

After a successful testing phase, the RFC is passed into the Deployment Approval phase. This phase requires a formal decision on the part of a deployment approver or a group of approvers that the required testing and information gathering activities have been successfully completed and the changes can move into deployment.

  • Select Request Deployment Approval to advance the ticket to the Deployment state.

  • Select Reject for Deployment to return the ticket to the Release Request state.
