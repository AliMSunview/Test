Release Management Workflow

The release management process is a sub-process of the change management life cycle. As a result, the process can only be started from an In-Progress RFC. After an RFC has been accepted, the person to whom the RFC was assigned starts the release management process.

Note: Release Management is included as a separate workflow that must be enabled by a ChangeGear Administrator.

The Release Process

The RFC Release status indicates that the release management process has started. In the Release stage, the release manager reviews the RFC to gain a broad understanding of the services and components affected and of the resources involved. This stage is useful to identify where more information is necessary.

Generally, a release manager takes ownership of the RFC when it enters the Release process.

To initiate the Release Management process:

  1. Navigate to the Change module.

  2. Open the appropriate In-Progress RFC.

  3. In the RFC, click on the Release Request option.

The RFC enters the Release stage and the RFC is evaluated for potential impact and roadblocks.

Note: If you do not see the Request Release option on the action bar, you may not have the correct privileges. Check with your ChangeGear Administrator.
