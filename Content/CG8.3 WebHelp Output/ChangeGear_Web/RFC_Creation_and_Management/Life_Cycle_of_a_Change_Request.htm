Change Request Life Cycle

ChangeGear uses a workflow status bar to represent each of the process stages a change request must go through before it is successfully implemented and closed. The work flow tracks the entire process from the submission of a change request (New) throughout the life cycle until the request is successfully closed (Closed).

The ChangeGear Request for Change (RFC) form provides several tools for reviewing and managing change requests. Each change request includes a record of the requester and the initial details of the request. If you decide that the change request doesn't meet the criteria for approval, you can use the RFC form to reject the request.

 Note: You may not have access to this ChangeGear feature depending on your permissions or ChangeGear edition.

The incident life cycle stages are based on the actions you perform:

  • Submitting a Request for Change

  • Approving RFCs

  • Assigned RFCs

  • Implementing RFCs

  • Reviewing the RFC

  • Change Pending Close

  • Closed RFC

Customizing the Change Request Process

The work flow described in this topic about the Life Cycle of a Change Request is based on a default process flow for a "typical" company. We understand your organization may have unique process needs and ChangeGear features fully configurable process flow capability.
