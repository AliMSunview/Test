How to Create and Submit a Change Request

You can submit a new change request through the Self-Service Portal by using the Request Change link. Some fields–such as Requester and Created–are automatically filled out when the form opens. Certain fields–marked in red–are required before the incident can be submitted or saved as a draft.

Note: You must click Submit to enter the change request in ChangeGear. If you save your request, no action by your IT department will be taken as the request has not been officially entered into ChangeGear.

To submit a change request:

  1. On the home page, click on the Request Change button.

  2. In the Summary field, enter a brief overview of your change.

  3. In the Description field, enter detailed information about your request.

  4. In the If Not Implemented field, enter a detailed description of the potential consequences of not completing the RFC

  5. In the Change Type list, select the type of change from the drop-down menu.

  6. Use the Priority drop-down menu to select the importance of the request.

  7. In the Due Date fields, enter the desired completion date and time.

  8. Attach any supporting documents, such as screen shots of error messages.

  9. Click Submit to create a new request with the details entered.

  10. If necessary, specify an Assignee and any Comments in the popup window that appears, then click OK to finish processing the ticket. You will receive an e-mail notification to confirm that your request was received.

To add more details to a change request:

  1. Click on the Active link under the Change Requests section of the Self-Service Portal home page. This brings you to the change module and displays your active change requests.

  2. Select the desired change in the grid and click the Edit button in the toolbar.

  3. Modify any fields that you wish to update.

  4. Click Save.

If you view your request later, the Workflow Status bar indicates what stage your request has entered, (e.g., New, In-Progress, Pending Close, Closed).
